# 메이크렌즈(MakeLens) 완벽 사용 가이드

## 메이크렌즈는 윈도우 전용 프로그램입니다.
혹시나 맥을 이용해서 가상머신을 사용시,
컴퓨터ID가 매번 변경되기때문에 두번째 접속시 인증키가 인증이 되지 않을 수 있으며
이때, 인증키를 재발송드리지 않으니 참고바랍니다.

## 개요
메이크렌즈(MakeLens)는 AI 기반 유튜브 영상 자동 제작 도구입니다.
대본 작성부터 음성 생성, 이미지 생성, 자막 편집, 최종 영상 출력까지
유튜브 영상 제작의 전 과정을 하나의 프로그램에서 자동으로 처리할 수 있습니다.

쇼츠(Shorts) 영상과 롱폼(Long-form) 영상 모두 제작할 수 있으며,
AI가 벤치마킹 분석, 주제 추천, 대본 생성, 이미지 생성, 음성 합성까지 도와줍니다.

메이크렌즈는 자동차의 역할을 하고, Vertex Ai API와 그록은 연료의 역할을 합니다.
처음에 API 세팅만 잘 해두면 이후에는 편하게 사용하실 수 있습니다.

## 프로그램 전체 구조

메이크렌즈는 크게 3개의 메인 메뉴와 4단계 작업 흐름으로 구성되어 있습니다.

### 메인 메뉴 (좌측 사이드바)
- 작업실: 영상 제작의 핵심 공간 (4단계 작업 흐름)
- 프로젝트: 저장된 프로젝트 관리 (불러오기/삭제)
- 설정: 스타일 프리셋 및 API 키 설정
- 버전 정보: 사이드바 중앙의 버전 번호를 클릭하면 현재 버전 확인 및 최신 버전 업데이트가 가능합니다.
- 딸깍 버튼: 사이드바에 있는 특별한 버튼으로, 모든 작업을 원클릭으로 자동화합니다.

### 4단계 작업 흐름 (작업실 내부)
1. 대본 - 벤치마킹 분석 → 주제 추천 → AI 대본 생성
2. 장면/이미지 - 장면 분석 → AI 이미지 생성 → 동영상 생성(그록 연동)
3. 음성 - AI 음성(TTS) 생성 및 미리듣기
4. 최종 영상 - 편집 준비 → 자막/효과 편집 → 영상 출력

## 다운로드 및 설치

### 다운로드
1. 튜브렌즈 홈페이지의 '프로그램 다운' 메뉴에서 '메이크렌즈'를 다운로드합니다.
2. 다운로드가 안 되는 경우, 본인이 사용하는 백신 프로그램(V3, 알약, 윈도우 디펜더 등)과 바이러스 및 위협방지의 '실시간 보호 기능'을 잠시 끄고 다운로드해 보세요.

### 설치
1. 다운로드한 설치 파일을 더블 클릭하여 설치합니다.
2. 컴퓨터에 1.5GB 이상의 여유 공간이 필요합니다.
3. 기본적으로 바탕 화면에 설치되게 되어 있습니다.
4. 나중에 생성된 이미지나 영상을 빠르게 사용하시려면 바탕 화면에 설치하는 것을 추천드립니다.
5. 설치가 완료되면 바탕 화면에 바로가기 아이콘과 프로그램 폴더가 생성됩니다.

### 설치 폴더 구성
메이크렌즈 폴더를 열면 다음과 같은 구성입니다:
- _internal 폴더: 프로그램 내부 파일 (수정하지 마세요)
- 메이크렌즈 오토(MakelensAuto) 폴더: 메이크렌즈와 연동되는 크롬 확장 프로그램 (동영상 생성용)
- 프로젝트 폴더: 생성된 프로젝트들이 저장되는 폴더
- 효과음 폴더: 효과음 파일을 넣는 폴더 (사용자가 직접 추가 가능)
- 배경음악 폴더: 배경 음악 파일을 넣는 폴더 (사용자가 직접 추가 가능)
- MakeLens.exe: 메이크렌즈 실행 파일
- uninstall.exe: 프로그램 삭제용 파일

### 메이크렌즈 오토(MakelensAuto) 용도
- 그록 영상 생성
- 위스크 이미지 생성
- 플로우(flow) 이미지 생성
- 썸네일(flow) 이미지 생성

참고 : 기존 그록오토(GrokAuto)가 메이크렌즈 오토(MakelensAuto)로 변경되었습니다.

## 처음 시작하기: 인증 및 API 설정

### 1단계: 인증키 입력
1. 메이크렌즈를 처음 실행하면 인증키 입력창이 나타납니다.
2. 무료 체험 인증키 또는 유료 인증키를 입력하고 확인을 누릅니다.
3. 인증이 성공적으로 완료되면 확인 메시지가 표시됩니다.

### 2단계: Vertex AI API 키 설정 (필수)
1. 인증 후 프로그램에 들어가면 "Gemini API 키 또는 Vertex AI JSON이 등록되지 않았습니다."라는 안내 메시지가 표시됩니다.
2. 확인을 누르면 자동으로 설정 탭으로 이동합니다.
3. 되도록 Vertex AI JSON을 받아서 등록합니다. (26.03.10 구글정책 변경)
4. API 설정 저장 버튼을 클릭하면 알림음과 함께 저장이 완료됩니다.

### $300 무료 크레딧 받는 방법
- 300달러(약 43만원) 무료 크레딧은 구글 클라우드 콘솔(https://cloud.google.com/cloud-console?hl=ko)에서 받을 수 있습니다. (정책 변경 가능)
- Vertex AI JSON 다운받고, 결제 계정 연결 및 Vertex AI API 사용 설정에서 '사용'까지 완료해야 메이크렌즈에서 사용 가능합니다.
- 자세한 발급 방법은 별도의 안내 영상을 참고해 주세요.
영상 링크 : https://youtu.be/tbU1uNAQL0Q

### 인증키 확인 및 관리
- 설정의 '인증키'탭에서 본인의 인증 상태와 남은 기간을 확인할 수 있습니다.
- 인증키 재설정: 무료 체험 기간 만료 전에 유료 인증키를 구매했다면, 인증키 재설정 버튼을 눌러 새 키를 입력할 수 있습니다.
- 1개의 인증키는 1개의 PC에서만 사용 가능합니다. (윈도우 전용 / 맥 사용불가 / 가상윈도우 불가)

### API 크레딧 및 사용량 확인
- 남은 크레딧 보기: 클릭하면 300달러 중 얼마나 남았는지 확인할 수 있습니다.
  - 크레딧이 안 보이면 우측 상단에서 해당 구글 계정과 프로젝트가 맞는지 확인하세요.
  - 또는 구글 콘솔의 '결제' 탭에서 확인하실 수 있습니다.

## 프로젝트 생성 및 관리

### 새 프로젝트 만들기
1. 화면 상단의 '프로젝트 이름' 입력란에 원하는 프로젝트 이름을 입력합니다.
2. 저장 버튼을 클릭하거나 Enter 키를 눌러 프로젝트를 저장합니다.
3. 프로젝트 이름을 먼저 저장해야 모든 기능을 사용할 수 있습니다.

### 새 프로젝트 초기화 (+ 버튼)
- 프로젝트 이름 옆의 + 버튼을 클릭하면 새 프로젝트를 시작할 수 있습니다.
- "새 프로젝트를 생성하시겠습니까?" 확인 팝업이 뜹니다.

### 프로젝트 저장
- 저장 버튼을 클릭하면 현재 작업 상태가 저장됩니다.
- 프로그램 종료 시에도 자동으로 마지막 상태가 저장됩니다.
- 일부 작업(이미지 생성, 음성 생성 등) 완료 후에는 자동 저장됩니다.

### 프로젝트 폴더 구조
각 프로젝트 폴더 안에는 다음과 같은 하위 폴더가 자동으로 생성됩니다:
- 이미지_및_오디오 폴더
  - 이미지: 생성된 이미지가 자동 저장됩니다. 이미지를 교체하면 여기도 자동으로 변경됩니다.
  - 오디오: TTS 음성 파일이 자동 저장됩니다.
  - 비디오: 그록에서 생성한 동영상이 저장됩니다.
- 최종영상 폴더: 최종 내보내기한 영상이 저장됩니다.
- 캐릭터 폴더: 캐릭터 일관성에 사용된 이미지가 저장됩니다.

### 프로젝트 탭 (사이드바 > 프로젝트)
- 저장된 프로젝트 목록을 확인할 수 있습니다.
- 프로젝트를 클릭하면 해당 프로젝트를 불러옵니다.
- 현재 프로젝트가 있는 경우 저장 확인 팝업이 표시됩니다:
  - 저장 후 전환: 현재 프로젝트를 저장한 후 전환
  - 저장 안함: 저장하지 않고 전환
  - 취소: 전환 취소
- 프로젝트 카드 우측의 삭제 버튼을 클릭하면 삭제 확인 후 프로젝트가 삭제됩니다.

### 프로젝트에 저장되는 정보
- 대본 텍스트 (벤치마킹 대본 포함)
- 벤치마킹 분석 결과 및 추천 주제 목록
- 장면 데이터 (나레이션, 프롬프트)
- 생성된 이미지
- 생성된 음성 파일
- 자막 데이터 및 자막 스타일 설정
- 이미지 효과, 트랜지션, 효과음 설정
- 스타일 프리셋 설정
- 캐릭터 이미지
- 음성 설정 (목소리, 말투, 속도 등)

## 1단계: 대본 (Script)

대본 탭에서는 유튜브 영상의 대본을 작성합니다.
두 가지 모드를 선택할 수 있습니다.

### 모드 선택

#### 벤치대본 모드 (기본)
조회수 높은 기존 대본들을 분석하여, 그 스타일을 벤치마킹한 새로운 대본을 생성합니다.
이 기능은 메이크렌즈의 핵심 기능 중 하나입니다.

#### 일반대본 모드
뉴스 기사, 블로그 포스트 등 원문을 입력하면 유튜브용 대본으로 변환합니다.

### 영상 형식 설정

- 쇼츠: 10초 이상의 세로 영상 (초 단위로 설정 가능)
- 롱폼: 1분~60분 분량의 가로 영상 (1분 단위로 설정 가능) / 퀄리티를 위해 30분 이내로 추천
- 형식에 따라 예상 글자 수가 자동 계산됩니다.

### 롱폼도 가능한가요?
메이크렌즈 공지 팝업에 안내드린것처럼
메이크렌즈는 구글의 Vertex AI api(gemini) 기반이며, 로컬 프로그램입니다. 
롱폼은 쇼츠와 달리 대본이 길수록, 장면이 많을수록, 이미지보다 비디오를 많이 사용할수록, 텍스트 에디터의 자막수가 많을수록, 
많은 api 요청이 따르고, 인터넷 환경과 컴퓨터 사양이 너무 중요합니다.

즉, 
1) 대본이 길어 여러차례 gemini 요청  중 3번의 재시도가 끊기면 실패로 돌아가기도하고, 
2) 와이파이로 인해 인터넷환경이 좋지 않을시(특히 그록), 에러가 생길수도 있으며,
3) 컴퓨터 사양이 낮은 CPU와 RAM 사용시, 시간이 엄청 오래걸리거나, 부하가 올 수 있습니다.
4) 또한 현재 방식이 텍스트에디터의 '각 자막줄'을 하나씩 png로 변경해서  영상에 씌우는 방식으로,
롱폼 경우, 자막줄이 많으면 상당히 오래걸릴 수 있습니다.

그래서 본인 컴퓨터에서 분량을 직접 조절해가며
체크해보셔야합니다 :)

그래서 모든 컴퓨터에 롱폼이 유연하게 동작하는것은
불가능합니다 :)

최대한 개선중이며, 위 내용을 꼭 인지하시고 스트레스가 덜 하시길 바라봅니다.

### 벤치대본 모드 사용법

#### 1. 벤치마킹 대본 입력
- 좌측 '벤치마킹 대본' 영역에 조회수 높은 대본들을 붙여넣습니다.
- 대본 3개 이상을 넣는 것을 권장합니다.
- 대본과 대본 사이에는 --- (하이픈 3개 이상)을 넣어 구분합니다.
- 대본파일 불러오기 버튼으로 txt 또는 xlsx 파일을 불러올 수도 있습니다.
  - txt 파일: ---로 구분된 대본 텍스트
  - xlsx 파일: '자막' 헤더가 있는 열의 데이터를 자동으로 읽어옵니다.
  - 튜브렌즈에서 자막 수집 후 엑셀 저장한 파일도 그대로 사용 가능합니다.

#### 2. 벤치 분석 및 주제 추천
- 벤치 분석 및 주제 추천 버튼을 클릭합니다.
- AI가 입력한 대본들의 후킹, 말투, 기승전결 등을 분석하고 지침으로 저장합니다.
- 분석이 끝나면 우측에 추천 주제 10개가 표시됩니다.
- 각 주제 옆에 "높음", "중간" 등 바이럴 가능성이 표시됩니다. 이는 AI가 추측하는 바이럴 정도입니다.

#### 3. 주제 방향 설정 (선택 사항)
- 우측 상단의 입력란에 원하는 방향을 입력할 수 있습니다.
- 예: "물건", "음식", "여행지", "IT기기", "회사" 등
- 벤치마킹 채널이 인물에 대해 다뤘더라도, 여기에 "음식"이라고 입력하면 같은 결(스타일)로 음식 관련 주제를 추천해 줍니다.
- 이를 통해 떡상 채널의 결을 파악해서 다른 주제로 파생시켜 나갈 수 있습니다.

#### 4. 추천 주제 활용
- 추천된 주제를 클릭하면 상세 정보 팝업이 표시됩니다:
  - 주제명
  - 메인 소재 (누구/무엇에 대한 내용인지)
  - 벤치 대본과의 유사점
  - 대본 작성 흐름 (어떤 흐름으로 대본을 만들 것인지)
  - 바이럴 가능성
- 대본 생성 버튼을 클릭하면 해당 주제로 AI 대본이 자동 생성됩니다.
- 주제를 우클릭하면 제목이 클립보드에 복사됩니다.
- 주제 10개 재추천 버튼으로 새로운 주제를 받을 수 있습니다.

#### 5. 대본 수정
- 생성된 대본이 마음에 들면 바로 진행해도 됩니다.
- 마음에 안 드는 부분이 있다면 대본 영역에서 바로 수정할 수 있습니다.
- 대본이 너무 짧으면 더 길게 쓰고, 너무 길면 삭제하여 편집합니다.

#### 6. 원본 대본으로 가공 (벤치대본 모드의 추가 기능)
- 벤치대본 모드에서 '원본 대본' 버튼을 클릭하면 빈 입력 공간이 나타납니다.
- 이곳에 가공하고 싶은 원본 대본을 넣으면, 벤치마킹 분석한 결(스타일)에 맞춰 유튜브형 대본으로 변경해 줍니다.
- 예시: 다른 주제의 대본을 넣으면, 벤치마킹 채널의 스타일로 해당 내용을 재구성해 줍니다.

> 팁: 같은 벤치마킹 프로젝트를 계속 운영하면서, 다양한 주제로 매번 영상을 생성하여 한 채널을 운영하시면 됩니다.

## 2단계: 장면/이미지 (Scenes/Images)

대본을 기반으로 장면을 분석하고, 각 장면에 맞는 AI 이미지를 생성합니다.

### 장면 분석

#### 장면 분석 실행
1. 먼저 스타일과 이미지 수를 설정합니다.
2. 장면 분석 실행 버튼을 클릭합니다.
3. AI가 대본을 토대로 배경, 스타일 등을 분석해서 장면을 분할하고, 각 장면에 맞는 이미지 프롬프트와 동영상 프롬프트를 자동 생성합니다.
4. 분석이 완료되면 장면 카드 목록이 표시됩니다.

> 중요: 스타일과 이미지 수를 먼저 정한 후에 장면 분석 실행을 해야 합니다.

#### 장면 분할 옵션
장면을 나누는 방식을 선택할 수 있습니다:
- 기본 (복합 분할): AI가 자동으로 최적의 장면 수를 결정합니다. 쇼츠 선택시만 선택 가능한 옵션이며, 거의 0.5문장~1문장마다 분할됩니다. 쇼츠는 빠른 변화가 있어야 이탈률이 적기 때문입니다.
- 전체 이미지 수 지정: 전체 이미지 개수를 직접 지정합니다 (예: 5장, 6장, 10장 내외). 대본을 지정한 수만큼의 문장으로 분할해서 이미지를 생성합니다.
- 문장 당 이미지 수 지정: 문장 몇 개마다 이미지 1장을 생성할지 지정합니다 (예: 1문장당, 2문장당).

#### 장면 분석 재실행
이미 결과가 있는 상태에서 장면 분석 실행을 다시 누르면:
- 전체 새로 생성: 분할부터 전부 다시 만듭니다.
- 이미지/동영상 프롬프트 재생성: 분할과 나레이션은 유지하고 프롬프트만 새로 생성합니다. 스타일을 바꿨을 때 유용합니다.

### 장면 카드

각 장면 카드에는 다음 정보가 표시됩니다:

- 나레이션: 해당 장면의 대본 텍스트
  - 나레이션 텍스트 안에서 Enter 키를 누르면 장면이 자동으로 분할됩니다.
- 이미지 프롬프트: AI가 이미지를 생성할 때 사용하는 설명
- 동영상 프롬프트: 그록에서 동영상 생성 시 사용하는 설명
- 이미지 미리보기: 생성된 이미지가 표시됩니다 (클릭하면 크게 볼 수 있음)
- 캐릭터 표시: 장면 번호 옆에 캐릭터 1, 2, 3이 같은 색상으로 표시되어 어떤 장면에 어떤 캐릭터가 들어가는지 바로 확인할 수 있습니다.
- 초록색 테두리: 이미지에 초록색 테두리가 있으면 해당 이미지의 동영상이 존재한다는 뜻입니다. 어디까지 영상이 생성되었는지 쉽게 확인할 수 있습니다.

#### 장면 편집 버튼
각 장면 카드에는 다음 버튼들이 있습니다:
- 합치기 (화살표): 아래 장면과 현재 장면을 합칩니다. 장면 2의 대본이 장면 1로 끌려오고 장면 2가 사라집니다.
- 삭제 (휴지통): 현재 장면을 삭제합니다.
- 프롬프트 재생성 (별표): 해당 장면 하나의 이미지 프롬프트와 동영상 프롬프트를 다시 생성합니다.
- 이미지 재생성 (회전 버튼): 현재 프롬프트로 해당 장면의 이미지를 다시 생성합니다.
- 영상 재생성 (플레이 버튼): 해당 장면 하나의 영상만 그록에서 새로 생성합니다.
- 이미지 교체/불러오기 : 이미지 영역에 원하는 이미지를 드래그앤드롭하여 넣거나, 마우스 우클릭하면 외부 이미지를 불러와 교체할 수 있습니다.

- 이어서 이미지 생성: 이미 나온 이미지를 제외하고 나머지만 이어서 생성합니다. 생성 도중 중단했을 때 유용합니다.

### 이미지 저장/백업

## 그록(Grok) 연동 동영상 생성

메이크렌즈는 그록(Grok) 사이트와 연동하여 이미지를 무료로 동영상으로 변환할 수 있습니다.

> 중요: 그록을 연동한 무료 영상 생성 기능은 그록의 정책 변경이나 차단에 의해 언제든지 사라질 수 있습니다. 
메이크렌즈의 메인 기능은 이미지를 생성하고 합쳐서 영상을 만드는 것이며, 그록 연동은 추가 기능입니다.
또한 인터넷 환경 또는 그록 사이트의 서버에 따라, 튕기는 현상이 발생할 수 있습니다. 
해당 프로젝트의 비디오 폴더에 저장이 되지 않고, 다운로드는 되었는데 확장앱에서 '삭제됨'이라고 뜰 경우,
이럴때, 크롬 브라우저에서 '설정' - '다운로드' - '위치'를 파악하여 저장 된 영상을 확인 하실 수 있으니 참고바랍니다.
이때, 일반적으로는 사용자 컴퓨터의 '다운로드' 폴더로 이동되곤 합니다.
- 그록 사이트가 만약 영문으로 되어있다면, 확장앱이 위치를 인식하지 못할 수 있으니, 영문페이지로 되어있다면, 반드시 한국어로 변경후 사용해주세요.


### 그록 연동 설정 방법 (확장앱)
- 사용법 영상 참조 : 링크 https://youtu.be/z5x_SbaaPi0?si=a5EaOvidARjH_Ha2&t=1339
- 메이크렌즈 오토(MakelensAuto) 확장앱은 그록 사이트에서 영상을 생성하고 받거나, 위스크에서 이미지생성을 할 때 반드시 필요합니다.

#### 1단계: 크롬 브라우저에서 그록 로그인
1. 반드시 크롬 브라우저를 사용해야 합니다 (웨일, 엣지 등 다른 브라우저 불가).
2. 크롬에서 그록(grok.com)에 접속합니다.
3. 로그인 버튼을 클릭하고 구글 계정으로 로그인합니다.

#### 2단계: 그록 설정 변경
1. 로그인 후 왼쪽 하단의 프로필 버튼을 클릭합니다.
2. 설정을 클릭합니다.
3. 세 번째에 있는 '동작' 버튼을 클릭합니다.
4. 하단에 있는 "자동 비디오 생성 활성화"를 반드시 꺼야 합니다.
   - 이 설정을 끄지 않으면 메이크렌즈에서 동영상 프롬프트를 자동으로 입력할 수 없습니다.

#### 3단계: 크롬 확장 프로그램 설치
1. 크롬 우측 상단의 점 세 개(더보기) 버튼을 클릭합니다.
2. '확장 프로그램' > '확장 프로그램 관리'를 클릭합니다.
3. 우측 상단의 개발자 모드 토글을 켭니다.
4. 왼쪽의 '압축 해제된 확장 프로그램을 로드합니다' 버튼을 클릭합니다.
5. 메이크렌즈 설치 폴더에서 'MakelensAuto' 폴더를 선택합니다 (폴더 자체를 선택, 파일이 아님).
6. '폴더 선택'을 클릭하면 "메이크렌즈 그록 비디오 제너레이터"가 설치됩니다.

#### 4단계: 확장 프로그램 고정
1. 크롬 우측 상단의 퍼즐 모양 아이콘을 클릭합니다.
2. '메이크렌즈 그록'을 찾아 핀 아이콘을 클릭하여 고정합니다.
3. 고정된 아이콘을 클릭하면 사이드바가 열립니다.
4. 메이크렌즈가 켜져 있으면 "연결됨"으로 표시됩니다. 꺼져 있으면 연결이 끊깁니다.

### 동영상 생성 사용법

#### 전체 영상 생성
1. 장면/이미지 탭에서 이미지가 모두 생성된 상태에서 이미지/영상 생성 > 전체 영상 생성을 클릭합니다.
2. 크롬의 사이드패널에서 이미지가 영상으로 생성되는 과정을 확인할 수 있습니다.
3. 상단에는 작업 대기열, 하단에는 완료된 장면이 표시됩니다.
4. 영상 생성이 완료되면 자동으로 업스케일링 후 저장됩니다.
5. 하나의 영상을 생성하는 데 보통 40~50초 정도 소요됩니다.

#### 장면별 영상 재생성
- 장면 카드의 플레이 버튼을 클릭하면 해당 장면 하나의 영상만 새로 생성할 수 있습니다.

#### 영상 존재 확인
- 이미지에 초록색 테두리가 있으면 해당 이미지의 동영상이 비디오 폴더에 존재한다는 뜻입니다.

> 주의: 그록은 하루 영상 생성 한도가 있습니다. 한도가 초과되면 새 구글 아이디로 로그인하거나 다음 날 다시 시도하세요.(또는 슈퍼그록 유료가입)

### 그록 확장앱에 '장면 에러'가 발생했을 경우, 아래 내용을 체크해보세요
- 와이파이 인터넷 : 메이크렌즈와 확장앱의 연결이 끊기면 장면에러가 발생할 수 있습니다. 되도록 유선랜을 사용하는 것을 권장드립니다.
- 메모리 부족 : 컴퓨터 사양이 낮으면 확장앱과의 연동성이 끊기는 경우가 발생할 수 있으니, 최대한 영상생성 시 다른작업을 멈춰주세요.
- 장면에러 또는 팅김 현상이 발생하는 것 같다면, '개별 영상생성'으로 재시도를 해보세요.

### 그록 확장앱 업데이트 방법
1) 크롬 브라우저의 우측 상단 점3개 클릭
2) 확장프로그램 - 확장프로그램 관리 클릭
3) 메이크렌즈 오토(MakelensAuto) 확장앱 새로고침 버튼 클릭 또는 삭제 후, 메이크렌즈 오토(MakelensAuto) 재설치
> 주의: 메이크렌즈 버전이 업데이트 되면, 확장앱도 반드시 업데이트 해주셔야합니다.

### 주의사항
- 그록 영상생성시, 1개의 크롬창만 띄워야합니다. 영상생성 중, 다른 크롬창을 띄우면 연동이 되지않습니다.
- rate limit 라고 그록에 뜬다면, 당신의 그록 오늘 할당량이 만료된것이니, 다른 계정으로 로그인하거나, 슈퍼 그록을 이용하세요.

### Q: 그록으로 영상생성이 안돼요.
A: 첫째, 현재 메이크렌즈와 그록 확장앱이 최신버전인지 확인해보세요. 프로그램 좌측 버전 숫자를 누르면 최신버전인지 확인이 가능합니다.
확장앱도 동일하게 새로운 버전으로 새로고침 또는 재설치 해주세요. 확장앱을 반드시 켜두어야합니다.
둘째, 그록 사이트 화면에 'rate limit'이라는 글자가 보이면, 오늘의 할당량이 만료된 것입니다.
새로운 계정으로 로그인하시거나, 슈퍼그록을 가입해서 사용하세요.

### Q: ❌ 장면 2 실패: 라디오 그룹을 찾을 수 없습니다. 이렇게 떠요 왜그런거죠?
A: 화면을 작게 만드신것 같습니다. 확장앱이 버튼을 찾아야하는데 화면을 좁게 만들어 버튼이 보이지 않는거에요.
그록이 열려있는 브라우저를 크게, 또는 최대화 해주세요.

## 3단계: 음성 (Voice/TTS)

AI를 활용하여 대본의 음성을 자동으로 생성합니다.

### 대본 확인 및 수정 (미러링)
- 상단에 대본이 미러링되어 표시됩니다.
- 여기서 대본을 수정하면 1단계 대본 탭에도 실시간으로 반영됩니다.
- 반대로 대본 탭에서 수정해도 여기에 자동으로 반영됩니다.

> 주의: 이곳에서 수정한 대본이 장면/이미지 탭의 나레이션과 호환되지는 않습니다. 나레이션 대본과 여기 대본이 일치하지 않으면 나중에 최종 영상에서 TTS 음성과 자막 타이밍이 안 맞을 수 있으니 주의하세요.

### 500자 분할 기능
- 대본 옆에 500자 분할 토글이 있습니다.
- 켜면 대본이 500자를 넘어갈 때 음성 생성을 분할해서 만듭니다.
- 구글TTS는 기능상 글자 수가 많을수록 음성이 깨지거나 갑자기 속삭이는 등 퀄리티가 낮아질 수 있기 때문에 500자씩 분할 생성 가능하도록 설계되어 있습니다.
- 분할 생성된 음성들은 알아서 자동으로 합쳐주니 걱정하지 않아도 됩니다.
- 단, 분할 생성 시 목소리가 조금씩 다를 수 있습니다.
- 목소리 퀄리티를 위해 항상 켜두고 사용하는 것을 권장합니다.

### 💡구글TTS 이용꿀팁 & 주의사항

- 음성 깨짐 방지: 장문 입력 시 발생하는 음성 왜곡(동굴 소리, 속삭임 등)을 막기 위해 500자 분할 토글을 켜고 생성하세요.
- 생성 시 차이 발생: 동일한 목소리 설정에서도 출력 시마다 목소리가 조금씩 다를 수 있는 점은 구글 TTS의 기술적 특성입니다.
- 토글 활용: 500자 분할 토글 설정을 변경해가며 여러 번 시도해 보시면 더 나은 결과물을 얻을 수 있습니다.
- 외부 음성 적용: 완벽한 일관성과 고품질 음원을 원하신다면 타 서비스에서 제작한 음성 파일을 업로드하여 사용해 보세요.

### 말투 스타일 (5종)
- 기본: 일반적인 말투
- 명랑하게: 밝고 경쾌한 말투
- 차분하게: 잔잔하고 침착한 말투
- 강하고 단호하게: 힘 있고 확신에 찬 말투
- 뉴스 리포터처럼: 전문적인 뉴스 앵커 말투

### 음성 속도
- 슬라이더로 0.5x ~ 2.0x 속도를 조절할 수 있습니다.
- 기본값: 1.1x (약간 빠르게)
- 구글 TTS에서는 원래 음성 속도 조절이 되지 않지만, 메이크렌즈에서 이를 가능하게 해두었습니다.
- 쇼츠의 경우 1.3~1.4배속을 추천합니다. 너무 루즈하면 이탈률이 높아질 수 있습니다.

### 말투 세부 지시 (디렉팅, 선택 사항)
- AI 음성의 발화 스타일을 더 세밀하게 지시할 수 있습니다.
- 위의 말투 스타일을 선택해도 좋고, 원하는 말투가 있다면 이곳에 직접 작성합니다.
- 예: "억양이 센 부산 사투리", "뉴스 앵커처럼 깔끔하고 단호하게, 3초마다 명확하게 끊어서 읽어줘" 등
- 작성한 내용대로 TTS에 말투 지시가 반영됩니다.

### AI 음성 생성
1. 목소리, 말투 스타일, 속도를 설정합니다.
2. AI 음성 생성 버튼을 클릭합니다.
3. 생성이 완료되면 하단에 오디오 플레이어가 나타납니다.

### 오디오 미리듣기
- 목소리 미리 듣기 : 샘플 목소리를 들을 수 있으나, 구글tts 특성상 같은 목소리여도 매번 톤이  다르니 참고바랍니다.
- 재생/일시정지, 탐색(시크바), 속도 조절(0.5x~2.0x), 볼륨 조절 기능이 있습니다.
- 현재/전체 재생 시간이 표시됩니다.

## 4단계: 최종 영상 (Editor)

생성된 대본, 이미지(또는 동영상), 음성을 합쳐 최종 영상을 만듭니다.

> 중요: 이미지를 수정했거나 대본을 수정해서 음성을 다시 만든 경우, '편집 준비'를 '반드시' 다시 눌러야 변경된 내용이 적용됩니다.

### 편집 준비

#### 편집 준비 버튼
클릭하면 설정 팝업이 표시됩니다:

##### 소스 선택
- 이미지 모드: 이미지를 기반으로 영상을 만듭니다. 트랜지션과 이미지 효과를 적용할 수 있습니다.
- 자동(영상우선) 모드: 그록에서 생성한 동영상과 이미지 조합으로 기반으로 영상을 만듭니다. 모든 장면에 영상이 있으면 영상기반으로, 영상이 없는 장면은 이미지를 조합하여 영상을 만듭니다.


##### 트랜지션
- 장면이 넘어갈 때 툭툭 끊기지 않고 부드러운 효과를 넣어 자연스럽게 전환됩니다.

##### 무음 자동 삭제
- TTS로 만든 음성의 중간중간 무음 구간을 자동으로 삭제해 줍니다.
- 쇼츠처럼 빠른 템포가 필요한 영상에 유용합니다.

##### 영상 효과음 보존
- 그록으로 생성한 영상의 소리가 편집된 영상에 포함시킬지 정할 수 있습니다.
- 비디오 볼륨의 50%로 설정되어 있습니다.

##### 이미지 효과
- 이미지에 모션 효과를 넣어 단순히 찍어낸 영상이 아닌 느낌을 줍니다.
- 유튜브 정책에 좀 더 안정적으로 영상을 만들 수 있습니다.
- 이미지 모드에서 자동 모드로 사용하는 것을 추천합니다.

#### 편집 준비 과정 (자동)
1. 음성 분석 (Whisper): 음성 파일을 위스퍼AI가 자막을 추출합니다.
2. 장면 매칭 (Gemini AI): 추출된 자막과 음성 타이밍 그리고 장면을 매칭합니다.
3. 자막 생성 (AI 분할): 장면별 자막을 자동으로 생성합니다.
(위스퍼AI가 TTS음성을 듣고 자막을 만들기 때문에, 오타가 있을 수 있습니다)

### 이미지 효과 (6종)

영상의 이미지에 적용할 수 있는 모션 효과입니다:

- 자동 모드 (Auto Mode): AI가 최적의 효과를 자동으로 배치합니다. 장면마다 다른 효과가 번갈아 적용됩니다. (추천)
- 줌 인/아웃 : 이미지가 확대/축소되는 효과로, 몰입감을 높여줍니다.
- 화면 이동 : 이미지가 가로 또는 세로로 이동하는 효과입니다.
- 유영 효과 : 이미지가 몽환적으로 부유하듯 움직이는 효과입니다.
- 복합 연출 : 줌과 패닝을 조합한 역동적인 연출입니다.
- 효과 없음 : 이미지를 정지 상태로 표시합니다.

### 비디오 캔버스 (자막 스타일 설정)

편집 준비가 완료되면 화면이 변하며 다양한 자막 스타일을 설정할 수 있습니다:

#### 텍스트 스타일
- B (굵게): 글씨를 굵게 합니다. 폰트에 따라 적용 여부가 다릅니다. 설정되면 청록색으로 표시됩니다.
- I (기울임): 글씨를 기울입니다.
- 폰트 선택: 무료 폰트들로, 마우스 휠로 빠르게 탐색할 수 있습니다.
  - 자주 쓰는 폰트는 별표를 눌러 즐겨찾기 해두면 편리합니다.
- 글씨 크기: 원하는 크기로 조절합니다.
  - 쇼츠 기본값: 90, 롱폼 기본값: 75

#### 색상 설정
- A (글자 색상): 자막 텍스트 색상을 변경합니다.
- 배경: 자막 배경 박스를 만들 수 있습니다. 투명도와 범위(패딩)도 조절 가능합니다.
- 테두리 (외곽선): 글자 테두리의 두께와 색상을 조절합니다.
- 그림자: 글자 그림자의 색상, 번짐, 흐림 등을 조절하여 예쁜 자막을 만들 수 있습니다.

#### 자막 위치
- 슬라이더로 자막의 세로 위치(%)를 조절합니다.
  - 쇼츠 기본값: 45%, 롱폼 기본값: 10%
  - 원하는 곳에 두고 설정하면 내보내기 시 해당 위치로 적용됩니다.

#### 한 줄 글자 수 (줄바꿈)
- 자막 한 줄에 몇 글자까지 놓이게 할지 정하는 설정입니다.
  - 쇼츠 기본값: 10자, 롱폼 기본값: 20자
- 슬라이더를 조절하면 글자가 늘어나거나 줄어듭니다.
- 슬라이더로 조정하면 어느 정도 단어별로 분할되지만, 옆에 있는 AI 분할 (반짝이 버튼)을 누르면 AI가 의미 단위로 자막을 더 자연스럽게 분할해 줍니다.

#### 자막 토글 (켜기/끄기)
- 자막 토글을 끄면 자막이 나오지 않는 영상을 만들 수 있습니다.
- 자막 없이 내보내기 하고 싶을 때 사용합니다.

### 효과음 설정

#### 효과음 선택
- 편집 화면에서 효과음 버튼을 클릭하면 효과음 목록이 표시됩니다.
- 음표 아이콘이 있는 것은 개발자가 넣어둔 기본 효과음입니다.
- 반짝이 표시가 있는 것은 사용자가 직접 넣은 커스텀 효과음입니다.

#### 커스텀 효과음 추가
- 메이크렌즈 폴더 안의 효과음 폴더에 원하는 효과음 파일을 넣으면 자동으로 메이크렌즈에서 연동됩니다.

#### 효과음 볼륨
- 효과음 볼륨을 조절할 수 있습니다 (보통 30% 정도 추천).
- 장면이 전환되는 타이밍에 효과음이 재생됩니다.

### 배경 음악 설정

#### 배경 음악 추가
- 메이크렌즈 폴더 안의 배경음악 폴더에 본인이 사용하려는 배경 음악 파일을 넣습니다.
- 메이크렌즈에서 배경음 버튼을 클릭하면 연동된 음악 목록이 표시됩니다.

#### 배경 음악 설정
- 목록에서 원하는 배경 음악을 선택합니다.
- 미리듣기가 가능합니다.
- 볼륨을 조절합니다 (보통 10% 정도 추천).
- 적용 버튼을 누르면 영상에 배경 음악이 반영됩니다.

### 자막 편집 (텍스트 에디터)

편집 준비가 완료되면 우측에 자막 목록이 표시됩니다:

#### 개별 자막 색상 변경
- 각 문장 우측의 팔레트 버튼을 누르면 해당 문장만 색상을 바꿀 수 있습니다.
- 다른 문장은 원래 색상 유지, 선택한 문장만 다른 색으로 표시됩니다.
- 원래대로 돌리려면 팔레트를 눌러 '기본값 사용'을 클릭하거나 Ctrl+Z로 되돌립니다.

#### 자막 타이밍 수정
- 각 자막의 시작/종료 시간(초)이 표시됩니다.
- 타이밍이 안 맞을 때 여기서 초를 직접 변경하면 됩니다.

#### 자막 분할/합치기
- 문장을 더 쪼개고 싶으면 원하는 곳에 커서를 놓고 Enter를 누르면 문장이 쪼개집니다.
- 다시 합치고 싶으면 문장 끝에 커서를 놓고 Delete 키를 누르거나 Ctrl+Z로 되돌립니다.

#### 자막 클릭 시 이동
- 자막을 클릭하면 해당 시점으로 미리보기가 이동합니다.

### 미리보기

- 좌측 프리뷰 영역에서 이미지와 자막이 실시간으로 미리보기됩니다.
- 화면을 마우스로 클릭해도 재생됩니다.
- Space 키로 재생/일시정지를 전환할 수 있습니다.
- 하단의 재생 버튼(▶)으로도 재생할 수 있습니다.
- 우측에서 배속 조절이 가능합니다 (빠르게/느리게 보기).
- 볼륨 조절도 할 수 있습니다.

### 영상 출력 (내보내기)

#### 내보내기 버튼
내보내기를 클릭하면 두 가지 옵션이 있습니다:

##### 전체 영상 내보내기
- 수정한 자막 세팅값으로 최종 영상을 MP4로 출력합니다.
- 쇼츠: 1080x1920 (세로) / 롱폼: 1920x1080 (가로) 해상도로 출력됩니다.
- 영상은 프로젝트 폴더 내 '완성영상' 폴더에 저장됩니다.
- 내보내기 완료 시 팝업이 표시됩니다.

##### SRT 자막 내보내기
- 다른 편집 프로그램(프리미어 프로, 캡컷 등)에서 자막을 수정하거나 다른 폰트를 사용하고 싶을 때 활용합니다.
- SRT 형식의 자막 파일로 내보냅니다.

#### QR코드 핸드폰 전송 기능
- 내보내기 완료 팝업에 QR코드가 표시됩니다.
- 핸드폰으로 QR을 찍으면 영상을 바로 다운로드할 수 있습니다.
- 중요: 핸드폰이 컴퓨터와 같은 Wi-Fi에 연결되어 있어야 전송이 가능합니다 (데이터로는 불가).
- 쇼츠의 경우 핸드폰에서 직접 업로드하면 첫 부분의 썸네일을 선택하기 편리합니다.
- 내보내기 완료 팝업의 폴더 열기 버튼으로 영상 저장 위치를 바로 확인할 수 있습니다.

#### QR 바로가기 버튼
- 내보내기 버튼 옆의 QR 모양 버튼을 누르면 마지막으로 내보내기한 영상의 QR코드와 폴더 열기가 다시 표시됩니다.
- 이전에 완성했던 영상을 찾을 때 유용합니다.

## 딸깍 버튼 (원클릭 자동화)

사이드바에 있는 딸깍 버튼은 모든 작업을 한 번에 자동화하는 기능입니다.

### 사용 조건
- 유튜브용 대본이 반드시 있어야 합니다.
- 대본이 없으면 딸깍 기능이 동작하지 않습니다.

### 딸깍 설정 화면

딸깍 버튼을 클릭하면 한 번에 모든 설정을 할 수 있는 화면이 나타납니다:

#### Step 1. 대본 및 영상 기초
- 영상 형식: 쇼츠 / 롱폼 선택 (영상 길이는 대본 길이에 맞게 자동 생성되므로 무시해도 됩니다)

#### Step 2. 비주얼 소스 연출
- 생성 모델: 나노바나나 프로 / 나노바나나 일반 / 이매진 4 울트라 / 위스크
- 이미지 스타일: 18종 스타일 프리셋 중 선택
- 이미지 수: 기본(복합 분할), 전체 이미지 수 지정 또는 문장당 이미지 수 지정
- 캐릭터 이미지: 캐릭터 사진 설정
- 동영상 생성 토글: 켜면 그록과 연동하여 이미지를 동영상으로 변환합니다. (크롬을 열고 확장앱을 열어두는 것을 권장드립니다.)

#### Step 3. AI 음성 및 나레이션
- 목소리 선택 : 남자/여자 목소리 선택
- 말투 스타일 선택
- 속도 설정 및 500자 분할 토글
- 말투 세부 지시

#### Step 4. 효과 및 자동화 범위
- 트랜지션 / 자막 / 무음삭제 / 영상효과음 사용여부
- 효과음 선택
- 배경 음악 선택
- 이미지 효과 및 영상 효과
- 자동화 범위
1) 1단계: 이미지 생성까지
2) 2단계: 음성 생성까지
3) 3단계: 미리보기까지
4) 4단계: 최종 MP4 내보내기까지

### 딸깍 실행
- 모든 설정을 마친 후 딸깍 시작 버튼을 누르면 자동화가 시작됩니다.
- 로딩 창이 뜨며 전체 경과 시간과 각 탭별 소요 시간을 확인할 수 있습니다.
- 중간에 중단하고 싶으면 중지 버튼을 누릅니다.
- 닫기 버튼을 누르면 로딩 창이 닫히고 각 탭에서 진행 상황을 직접 볼 수 있습니다.
- 다시 딸깍을 누르면 로딩 창을 다시 확인할 수 있습니다.

> 참고: 컴퓨터 사양에 따라 속도 차이가 있습니다. 특히 미리보기 영상과 최종 내보내기 영상은 CPU를 많이 사용하므로 사양이 낮으면 시간이 오래 걸릴 수 있습니다.

> 예시: 나노바나나 프로로 6장 이미지 + 그록 6개 동영상 + TTS + 최종 MP4 내보내기까지 약 10분 소요.

## 설정

### 스타일 탭

#### 스타일 프리셋 선택
- 18가지 스타일 프리셋이 이미지 썸네일과 함께 표시됩니다.
- 클릭하면 하단에서 스타일 프롬프트를 확인할 수 있고 텍스트를 수정하여 사용하는 것도 가능합니다.
- 선택한 스타일은 이미지 생성 시 적용됩니다.

#### 커스텀 스타일
- '커스텀' 카드를 클릭하여 직접 스타일을 정의할 수 있습니다.
- 이미지로 분석: 이미지를 업로드하면 AI가 해당 이미지의 스타일을 분석하여 프롬프트를 자동 생성합니다.

> 예시: 유행하는 해골 콘텐츠를 만들고 싶다면, 해당 스타일의 영상을 캡처하여 커스텀에 넣으면 AI가 스타일을 분석하여 비슷한 이미지를 생성할 수 있습니다.

> 참고: 커스텀 스타일 이미지 분석은 프로젝트가 먼저 생성되어야 사용할 수 있습니다.

### API 설정 탭

#### Vertex AI API Json 등록
- Google의 Vertex AI API를 사용하기 위한 Json 파일을 등록합니다.
- API 키가 없으면 대본 생성, 이미지 생성, 음성 생성 등의 AI 기능을 사용할 수 없습니다.
- API json 파일은 구글콘솔에서 발급받을 수 있습니다.
- 메이크렌즈는 반드시 유료API를 이용하시기 바랍니다.
참고 영상링크 : https://youtu.be/tbU1uNAQL0Q

#### 인증 관리
- 우측에서 인증 상태와 남은 기간을 확인할 수 있습니다.
- 인증키 재설정으로 새 키를 입력할 수 있습니다.

## 외부 콘텐츠 호환 (이미지/음성/영상 교체)

메이크렌즈는 다른 곳에서 만든 이미지, 영상, TTS 음성을 호환할 수 있습니다.

### 이미지 교체
- 프로젝트 폴더 > 이미지_및_오디오 > 이미지 폴더에서 파일명을 동일하게 맞춰 교체합니다.
예시) scene_1.png (scene_01.png X)
- 또는 장면 카드에서 마우스 우클릭으로 외부 이미지를 불러올 수 있습니다.

### 음성 교체
- 다른 TTS 서비스(일레븐랩스, 타입캐스트 등)에서 생성한 음성을 사용하고 싶다면:
- 음성 교체/불러오기 버튼을 눌러주세요.
- wav 또는 mp3로 교체/불러오기 가능합니다.

### 영상 교체
- 그록 이외의 영상 생성 서비스에서 만든 영상을 사용하고 싶다면:
- 프로젝트 폴더 > 이미지_및_오디오 > 비디오 폴더에서 기존 영상 파일명과 동일하게 교체합니다.
예시) scene_01_grok.mp4 (scene_1_grok.mp4 X)

> 중요: 이미지, 음성, 영상을 교체한 후에는 반드시 편집 준비를 다시 실행해야 변경 사항이 미리보기 및 내보내기에 적용됩니다.

## 에러 메시지 안내

### 400대 에러

#### 401 에러
- 원인: API 키가 올바르지 않습니다.
- 해결: 본인의 Vertex AI API가 올바르게 입력되었는지 재확인하세요.

#### 403 에러
- 원인: 권한 부족 또는 결제 미등록 상태입니다.
- 해결: 반드시 '결제 계정 연동'과 'Vertex AI API'가 활성화 된 유료 API 키를 사용해야 합니다.

#### 404 에러
- 거의 발생하지 않습니다.

#### 429 에러
- 원인: 사용량 한도 초과입니다. 분당 할당량 또는 하루 할당량을 모두 사용했다는 뜻입니다. (주의: 여기서말하는 분당 할당량은 1분당이 아니라 N분당입니다. 1분일수도 5분일수도 15분일수도 있습니다)
- 해결 (이미지 생성 시): 잠시 후 다시 시도하거나 내일 다시 시도하세요. 만약 설정의 유료 테스트 시, 되지않는다면 vertex ai api 영상을 보고 다시 설정해보세요. 이미지생성은 반드시 유료 api를 사용해야합니다.
- 해결 (그록 영상 생성 시): 오늘의 영상 생성 한도가 초과되었습니다. 새 구글 아이디로 로그인하거나 다음 날 다시 시도하세요. (또는 슈퍼그록 유료가입)

### 500대 에러

#### 500 에러, 503 에러, 타임아웃 에러
- 원인: 구글 서버 과부하 또는 오류입니다.
- 해결: 프로그램의 문제가 아니라 나노바나나 프로를 사용하는 사용자가 많아 구글 서버에 과부하가 일어난 것입니다.
  - 나노바나나 일반 또는 이매진 4 울트라로 전환하면 문제없이 생성 가능합니다.
  - 또는 시간을 두고 나중에 재시도하세요. 세계적으로 나노바나나 프로 사용자가 몰려서 그렇습니다.

### Safety 에러 (콘텐츠 정책 위반)
- 원인: 이미지 또는 영상 생성 시 콘텐츠 정책에 위반되는 내용이 감지되었습니다.
- 해결: 프롬프트를 수정하여 다시 시도하면 문제없이 생성 가능합니다.

> 참고: 위의 에러들은 메이크렌즈 자체의 문제가 아니므로, 개발자에게 별도로 연락할 필요가 없습니다.

## 자주 묻는 질문 (FAQ)

### Q. vertex(버텍스) 등록했는데, 유료 테스트하면 안된다고 나와요.
A. 우선 다시 영상을 참고하여 3,4단계가 잘되었는지 확인해보세요.
3단계 - 결제 계정 연결
4단계 - 'Vertex AI API'를 활성화 '사용'으로 했는지
그래도 안되면, 스트레스 받지 마시고, 새로운 계정/새로운 번호/새로운 카드로 처음부터 하시는걸 추천합니다.

### Q: 프로그램을 처음 실행했는데 아무것도 안 됩니다. 어떻게 해야 하나요?
A: 먼저 다음 세 가지를 확인해주세요:
   1. 인증키를 입력했는지 확인
   2. 프로젝트 이름을 입력하고 저장 버튼을 눌러 프로젝트를 생성했는지 확인
   3. 설정 > API 설정 탭에서 Vertex AI API Json을 등록했는지 확인
   이 세 가지가 완료되어야 모든 기능을 사용할 수 있습니다.
(주의 : 간혹 '메이크렌즈 인증키'와 '튜브렌즈 인증키'를 혼동하거나 'API키'와 혼동하는 경우가 있습니다. 튜브렌즈와 별도의 프로그램이오니 재확인 바랍니다)

### Q: Vertex AI API Json은 어디서 받나요?
A: 구글 콘솔 (https://cloud.google.com/free) 에서 발급받을 수 있습니다. 
- 300달러 무료 크레딧도 구글 클라우드 콘솔(https://cloud.google.com/free)에서 받을 수 있습니다. (정책 변경 가능).

### Q: 남은 API 크레딧은 어떻게 확인하나요?
A: 설정 > API 설정 탭에서 '남은 크레딧 보기' 버튼을 클릭하면 300달러 중 얼마나 남았는지 확인할 수 있습니다. 크레딧이 보이지 않으면 구글 계정과 프로젝트가 맞는지 확인하세요.
구글 콘솔의 '결제' 탭에서도 확인하실 수 있습니다.

### Q: 503 에러가 자주 발생합니다.
A: 나노바나나 프로 사용 시 구글 서버 과부하로 발생하는 문제입니다. 시간을 두고 나중에 시도하거나, 나노바나나2 또는 다른 모델로 전환하면 수월하게 생성할 수 있습니다.

### Q: "대본 생성에 실패했습니다" 오류가 나옵니다.
A: 다음 사항을 확인해보세요:
   - Vertex AI API JSON 파일이 올바르게 등록되어 있는지 확인
   - 인터넷 연결이 정상적인지 확인
   - API 할당량이 초과되지 않았는지 확인 (24시간 후 자동 복구)

### Q: 벤치마킹 대본은 어떻게 구분하나요?
A: 대본과 대본 사이에 --- (하이픈 3개 이상)을 넣어 구분합니다. 수동 입력 시에도 동일합니다.
   예시:
   ```
   첫 번째 대본 내용...
   ---
   두 번째 대본 내용...
   ---
   세 번째 대본 내용...
   ```

### Q: 쇼츠와 롱폼의 차이가 뭔가요?
A:
   - 쇼츠: 유튜브 쇼츠용 세로 영상 (1080x1920), 10초 이상
   - 롱폼: 일반 유튜브 영상용 가로 영상 (1920x1080), 1~60분
   이미지 생성시 프로 자동 모드에서 쇼츠는 9:16, 롱폼은 16:9 비율로 자동 생성됩니다.

### Q: 이미지 스타일은 중간에 바꿀 수 있나요?
A: 네, 설정 탭이나 장면/이미지 탭 상단의 스타일 드롭다운에서 언제든지 변경할 수 있습니다. 단, 변경 후에는 장면 분석 실행에서 '이미지/동영상 프롬프트 재생성'을 하거나 이미지를 다시 생성해야 새 스타일이 적용됩니다.

### Q: 이미지가 마음에 안 들어요. 특정 장면만 다시 만들 수 있나요?
A: 네, 해당 장면 카드에서 이미지 재생성 (회전 버튼)을 클릭하면 현재 프롬프트로 해당 장면의 이미지만 다시 생성됩니다. 프롬프트를 수정한 후 재생성하면 다른 느낌의 이미지를 얻을 수 있습니다.

### Q: 프롬프트를 직접 수정해도 되나요?
A: 네, 각 장면 카드의 '이미지 프롬프트'를 직접 수정할 수 있습니다. 수정 후 이미지 재생성 버튼으로 새 이미지를 생성하세요. 또는 프롬프트 재생성 버튼으로 대본에 맞게 프롬프트를 다시 자동 생성할 수도 있습니다.

### Q: 캐릭터 일관성은 어떻게 유지하나요?
A: 장면/이미지 탭 상단의 캐릭터 버튼(char_1, char_2, char_3)에 캐릭터 이미지를 업로드하세요. 해당 캐릭터가 등장하는 모든 장면에서 일관된 외모로 이미지가 생성됩니다.

### Q: 음성 생성 후 목소리가 마음에 안 들면 어떻게 하나요?
A: 목소리, 말투 스타일, 속도를 변경한 후 AI 음성 생성 버튼을 다시 클릭하면 새로운 음성이 생성됩니다.

### Q: TTS 음성이 중간에 속삭이거나 깨집니다.
A: 글자 수가 많으면 TTS 품질이 떨어질 수 있습니다. 500자 분할 토글을 켜면 500자씩 나누어 생성한 후 자동으로 합쳐주므로 품질이 향상됩니다.

### Q: 다른 TTS 서비스의 음성을 사용할 수 있나요?
A: 네, 1.4.8 버전 이후, '음성 교체/불러오기' 버튼을 눌러서, wav 또는 mp3 파일을 교체/불러오기 가능합니다. 교체후엔 반드시 편집 준비를 다시 실행하세요.

### Q: 자막이 자연스럽게 끊기지 않아요.
A: 4단계 '최종 영상' 탭에서 AI 분할 (반짝이 버튼)을 클릭하면 AI가 의미 단위로 자막을 다시 분할해 줍니다. '한 줄 글자 수' 슬라이더를 조절한 후 재분할하면 더 나은 결과를 얻을 수 있습니다. 또는 '편집준비'를 재실행해보세요.

### Q: 자막 위치를 조절하고 싶어요.
A: 4단계 '최종 영상' 탭에서 '자막 위치' 슬라이더를 조절하면 됩니다. 값이 작을수록 화면 아래쪽, 클수록 화면 위쪽에 자막이 표시됩니다.

### Q: 자막 없이 영상을 만들 수 있나요?
A: 네, 최종 영상 탭에서 자막 토글을 끄면 자막이 없는 영상으로 내보내기됩니다.

### Q: 특정 자막만 색상을 바꿀 수 있나요?
A: 네, 자막 에디터에서 해당 문장 우측의 팔레트 버튼을 클릭하면 그 문장만 다른 색상으로 변경할 수 있습니다.

### Q: 그록 연동이 안 됩니다.
A: 다음 사항을 확인해주세요:
   1. 크롬 브라우저를 사용하고 있는지 확인 (다른 브라우저 불가)
   2. 그록 사이트에 로그인이 되어 있는지 확인
   3. 그록 설정 > 동작 > 자동 비디오 생성 활성화가 꺼져 있는지 확인
   4. 크롬 확장 프로그램에 메이크렌즈 오토(MakelensAuto)가 설치되어 있는지 확인
   5. 메이크렌즈가 켜져 있고 사이드패널에 "연결됨"으로 표시되는지 확인

### Q: 그록 영상 생성시 화면 전환 실패:textarea를 찾을수 없다고 나오는데 어떻게 해야하나요?
1) 현재 버전을 체크해보세요. 현재 버전 + 확장앱도 새로고침/재설치로 현재 버전을 확인해주세요
2) 그록 사이트 rate limit이 뜨면, 오늘의  할당량이 만료된것입니다. 다른 계정으로 시도해주세요.
3) 와이파이보다는 유선랜을 사용해주세요. 와이파이 사용시 메이크렌즈와 확장앱의 연결이 불안정할수 있습니다.


### Q: 그록 영상 생성이 갑자기 안 됩니다.
A: 그록의 하루 영상 생성 한도가 초과되었을 수 있습니다. 새 구글 아이디로 로그인하거나 다음 날 다시 시도하세요. 또한 그록의 정책 변경으로 기능이 제한될 수도 있습니다.

### Q: 이미지 모드와 자동 모드의 차이가 뭔가요?
A: 편집 준비 시 선택할 수 있습니다.
   - 이미지 모드: 이미지를 기반으로 영상을 만들며, 이미지 효과(줌, 패닝 등)를 적용할 수 있습니다.
   - 자동(영상우선) 모드: 그록에서 생성한 동영상과 이미지 조합으로 기반으로 영상을 만듭니다. 모든 장면에 영상이 있으면 영상기반으로, 영상이 없는 장면은 이미지를 조합하여 영상을 만듭니다.

### Q: 효과음이나 배경 음악을 직접 추가할 수 있나요?
A: 네, 메이크렌즈 설치 폴더의 효과음 폴더에 효과음 파일을, 배경음악 폴더에 배경 음악 파일을 넣으면 자동으로 프로그램에 연동됩니다.

### Q: 편집 준비는 왜 해야 하나요?
A: '편집 준비'는 다음을 자동으로 처리합니다:
   - 생성 된 이미지와 영상을 랜더링
   - 음성 파일을 위스퍼AI가 자막생성
   - 대본과 음성 타이밍 그리고 장면의 매칭 (Gemini AI)
이미지나 음성을 수정한 후에는 반드시 편집 준비를 다시 실행해야 합니다.

### Q: 프로젝트 파일은 어디에 저장되나요?
A: 프로젝트 파일은 메이크렌즈 설치 폴더 내 '프로젝트' 폴더에 저장됩니다. 각 프로젝트별로 하위 폴더가 생성되며, 이미지, 오디오, 비디오, 완성 영상 등이 체계적으로 정리되어 있습니다.

### Q: 장면을 합치거나 분할할 수 있나요?
A: 네, 가능합니다:
   - 분할: 나레이션 텍스트 안에서 원하는 위치에 커서를 놓고 Enter를 누르면 장면이 자동으로 분할됩니다.
   - 합치기: 장면 카드의 합치기 버튼(화살표)을 클릭하면 아래 장면과 합쳐집니다.

### Q: 완성된 영상을 핸드폰으로 보내고 싶어요.
A: 내보내기 완료 후 팝업에 표시되는 QR코드를 핸드폰으로 스캔하면 바로 다운로드됩니다. 단, 컴퓨터와 같은 Wi-Fi에 연결되어 있어야 합니다.

### Q: 딸깍 버튼은 뭔가요?
A: 대본부터 최종 영상까지 모든 과정을 한 번에 자동으로 처리하는 원클릭 자동화 기능입니다. 설정만 해두고 딸깍 시작을 누르면 됩니다.

## 프로그램 업데이트 및 삭제

### 업데이트
- 프로그램 시작 시 자동으로 업데이트를 확인합니다.
- 사이드바 중앙의 버전 번호를 클릭하면 수동으로 최신 버전을 확인하고 업데이트할 수 있습니다.
- 업데이트 시에도 배경음악, 프로젝트, 효과음 데이터는 삭제되지 않으니 걱정하지 않으셔도 됩니다.
- 업데이트 후 프로그램 실행 시 공지 팝업에 업데이트 내역을 확인할 수 있습니다.

### 삭제
- 설치 폴더의 uninstall.exe 파일을 실행하여 프로그램을 삭제합니다.
- 삭제하더라도 배경음악, 프로젝트, 효과음 데이터는 삭제되지 않습니다.

## 문제 해결

### 업데이트가 되지 않는 경우 (업데이트 오류)
- 본인이 사용하는 백신 프로그램(V3, 알약, 윈도우 디펜더 등)과 바이러스 및 위협방지의 '실시간 보호 기능'을 잠시 끄고 다운로드해 보세요.
- 바이러스 위협방지의 '실시간 보호'를 끄고 실행해보세요.
- 수동으로 홈페이지의 프로그램 다운로드 페이지에서 받으실 수 있습니다.

### 다운로드가 되지 않는 경우
- 본인이 사용하는 백신 프로그램(V3, 알약, 윈도우 디펜더 등)과 바이러스 및 위협방지의 '실시간 보호 기능'을 잠시 끄고 다운로드해 보세요.

### 모든 기능이 비활성화되어 있는 경우
- 프로젝트 이름을 입력하고 저장 버튼을 클릭하여 프로젝트를 먼저 생성하세요.
- 프로젝트가 생성되어야 모든 입력 필드와 버튼이 활성화됩니다.

### API 관련 오류가 발생하는 경우
- 설정 > API 설정 탭에서 Vertex AI API Json 파일이 올바르게 등록되어 있는지 확인하세요.
- API 할당량이 초과된 경우 24시간 후에 자동으로 복구됩니다.
- 네트워크 연결이 불안정한 경우 다시 시도하세요.

### 이미지 생성이 실패하는 경우
- Vertex AI API Json 파일 확인
- 인터넷 연결 확인
- 프롬프트에 부적절한 내용(Safety 위반)이 포함되어 있지 않은지 확인
- 503 에러 시: 구글 서버 과부하. 시간을 두고 재시도하거나 나노바나나 2 또는 다른 모델로 전환
- 잠시 후 다시 시도 (API 서버 일시적 장애 가능)

### 음성 생성이 실패하는 경우
- 대본이 비어있지 않은지 확인 (1단계 대본 탭에서 대본 작성 필요)
- API 키 확인
- 대본이 너무 긴 경우 500자 분할 토글을 켜고 시도

### MP4 영상 생성이 실패하는 경우

### 프로그램이 느리게 동작하는 경우

## 롱폼 대본 생성 방식 안내

10분 이상의 긴 대본은 AI가 한 번에 쓰지 않습니다. 사람이 글을 쓰듯, 구조를 먼저 잡고 파트별로 나눠서 씁니다.

생성 과정:
1. AI가 먼저 전체 대본의 뼈대를 설계합니다 (후킹 → 전개 → 클라이맥스 → 결론)
2. 분량에 따라 자동으로 파트 수가 정해집니다
3. 각 파트를 순서대로 작성하며, 이전에 쓴 내용을 보면서 다음 파트를 이어 씁니다
4. 모든 파트가 완성되면 하나로 합쳐집니다

분량별 파트 수:
- 10분: 5파트
- 20분: 5파트
- 30분: 5파트
- 40분: 7파트
- 50분: 8파트
- 60분: 10파트

파트가 많을수록 생성 시간이 길어지지만, 
최대한 촘촘하고 밀도있는 대본이 만들어질 수 있도록 노력했습니다.

## 캐릭터 자동 생성 
: 캐릭터 자동 생성 (AI 캐릭터 이미지 생성)

■ 한줄 설명:
  대본에 등장하는 캐릭터들의 레퍼런스 이미지를 AI가 자동으로 만들어주는 기능입니다.
  이 이미지를 기반으로 장면마다 일관된 캐릭터 외형이 유지됩니다.

■ 어디에 있나요?
  - "장면/이미지" 탭 → 상단의 [🎨 이미지/영상 생성 ▼] 버튼 클릭 → [👤 캐릭터 이미지 생성] 선택
  - [전체 이미지 생성] 시 , 사용자 선택

■ 사용 조건:
  - 대본 분석(장면 분석)이 먼저 완료되어 있어야 합니다
  - Vertex AI API Json 파일이 등록되어 있어야 합니다

■ 사용 방법:

  [방법 1] 수동으로 캐릭터 생성하기
  ─────────────────────────────────
  1. "장면/이미지" 탭으로 이동합니다
  2. 상단의 [🎨 이미지/영상 생성 ▼] 버튼을 클릭합니다
  3. [👤 캐릭터 이미지 생성]을 선택합니다
  4. AI가 자동으로 대본에 필요한 캐릭터들의 이미지를 생성합니다. (퀄리티를 위해 나노바나나2 사용)
  5. 완료되면 상단 캐릭터 버튼에 생성된 이미지가 표시됩니다.

  [방법 2] 전체 이미지 생성 시 자동으로 함께 생성
  ──────────────────────────────────────────────
  1. [🔄 전체 이미지 생성]을 클릭합니다
  2. 아직 만들지 않은 캐릭터가 있으면 "캐릭터 이미지가 없습니다. 생성할까요?" 팝업이 뜹니다
  3. [✅ 예]를 누르면 캐릭터를 먼저 생성한 뒤 장면 이미지를 이어서 생성합니다
  4. [⏭️ 아니오]를 누르면 캐릭터 없이 바로 장면 이미지를 생성합니다

  [방법 3] 직접 이미지 넣기 (수동 업로드)
  ──────────────────────────────────────
  1. 상단의 캐릭터 버튼 [👤]을 직접 클릭합니다
  2. 파일 선택 창이 뜨면 원하는 캐릭터 이미지를 선택합니다
  3. 내가 직접 준비한 이미지를 캐릭터로 사용할 수 있습니다

■ 캐릭터 버튼 사용법:
  - 이미지가 없을 때: 👤 아이콘 (색깔 테두리)으로 표시됩니다
  - 이미지가 있을 때: 작은 썸네일로 표시됩니다
  - 왼쪽 클릭: 이미지 파일을 직접 선택할 수 있습니다
  - 오른쪽 클릭: 현재 이미지를 삭제합니다
  - 마우스 올리기: 큰 미리보기가 팝업으로 뜹니다

■ 참고 사항:
  - 이 기능으로 인해 더욱 일관성/통일성이 높은 이미지 생성이 가능해집니다.
  - 캐릭터는 AI 판단으로 대본에 필요한 인물을 생성합니다.
  - 캐릭터 이미지는 프로젝트에 자동 저장됩니다
  - 캐릭터가 마음에 들지 않으면 다시 생성하거나 직접 이미지를 넣을 수 있습니다
  - 여러 캐릭터가 있으면 동시에 생성됩니다 (최대 3개 병렬 처리)

■ 자주 묻는 질문:

  Q: 캐릭터 이미지를 바꾸고 싶어요
  A: 캐릭터 버튼을 왼쪽 클릭하면 새 이미지를 선택할 수 있고, 오른쪽 클릭하면 삭제 후 다시 생성할 수 있습니다.

  Q: 캐릭터가 4명 이상인데 3개밖에 안 돼요
  A: UI상에는 최대 3명까지 확인이 가능하고, 4명째 부터는 메모리에서 작동되며, 사진은 동일하게 자동 저장됩니다.

  Q: 생성된 캐릭터가 대본 내용과 다르게 나왔어요
  A: AI가 대본의 캐릭터 설명을 기반으로 생성하지만 완벽하지 않을 수 있습니다. 마음에 들지 않으면 직접 이미지를 업로드하거나 다시 생성해보세요.

## 비디오 미리보기
: 비디오 영상 재생

■ 한줄 설명:
  Grok으로 생성한 영상(Image-to-Video)을 장면 탭에서 바로 재생해볼 수 있는 기능입니다.
  각 장면의 이미지 위에 동그란 ▶ 플레이 버튼이 표시되며, 클릭하면 영상이 재생됩니다.

■ 어디에 있나요?
  "장면/이미지" 탭 → 각 장면 카드의 이미지 위에 동그란 ▶ 플레이 버튼
  (Grok 비디오가 생성된 장면에만 표시됩니다)

■ 사용 조건:
  - 해당 장면의 Grok 비디오(Image-to-Video)가 생성되어 있어야 합니다

■ 플레이 버튼이 보이는 조건:
  - 장면 이미지가 있고, 해당 장면의 Grok 비디오 파일이 존재할 때만 표시됩니다
  - 비디오가 있는 장면은 이미지 테두리가 초록색으로 바뀝니다

■ 사용 방법:

  [영상 재생하기]
  ──────────────
  1. "장면" 탭에서 Grok 비디오가 생성된 장면을 찾습니다
     (이미지 중앙에 동그란 ▶ 버튼이 보입니다)
  2. ▶ 버튼을 클릭합니다
  3. 전체 화면 오버레이 팝업이 뜨면서 영상이 자동 재생됩니다

  [재생 중 조작하기]
  ─────────────────
  - 재생/일시정지: 영상 화면을 클릭하거나 스페이스바를 누릅니다
  - 원하는 위치로 이동: 하단 슬라이더를 드래그합니다
  - 재생 속도 변경: 속도 버튼을 클릭합니다 (0.5배 ~ 2배속)
  - 볼륨 조절: 볼륨 슬라이더로 조절합니다
  - 닫기: 영상 바깥 배경을 클릭하거나 ESC 키를 누릅니다

■ 이미지 클릭 vs 플레이 버튼 클릭:
  - ▶ 버튼 클릭 → 비디오 재생 팝업이 뜹니다
  - ▶ 버튼 바깥 이미지 클릭 → 이미지 확대보기가 뜹니다
  - 오른쪽 클릭/드래그 → 이미지 불러오기

■ 참고 사항:
  - 비디오가 없는 장면에는 ▶ 버튼이 표시되지 않습니다
  - ▶ 버튼을 눌렀는데 "영상 파일이 없습니다" 메시지가 뜨면
    해당 장면의 Grok 비디오가 삭제되었거나 경로가 변경된 것입니다
  - 재생 팝업은 전체 화면 오버레이 형태로 뜹니다 (반투명 검정 배경)

■ 자주 묻는 질문:

  Q: 이미지에 ▶ 플레이 버튼이 안 보여요
  A: 해당 장면의 Grok 비디오를 먼저 생성해야 합니다. 장면 카드에서 Grok 영상 생성을 실행해주세요.

  Q: ▶ 버튼을 눌렀는데 "영상 파일이 없습니다"라고 떠요
  A: 비디오 파일이 삭제되었거나 프로젝트 폴더가 이동된 경우 발생할 수 있습니다. Grok 비디오를 다시 생성해주세요.

  Q: 영상이 재생되지 않거나 검은 화면만 나와요
  A: 비디오 파일이 손상되었을 수 있습니다. 해당 장면의 Grok 비디오를 다시 생성해보세요.

  Q: 재생 팝업을 어떻게 닫나요?
  A: 영상 바깥의 어두운 배경을 클릭하거나, 키보드의 ESC 키를 누르면 닫힙니다.

## SRT 자막 불러오기
: SRT 외부 자막 가져오기

■ 한줄 설명:
  외부에서 만든 SRT 자막 파일을 불러와서 현재 프로젝트의 자막을 교체하는 기능입니다.
  직접 자막을 수정했거나, 다른 도구에서 만든 자막을 사용하고 싶을 때 유용합니다.

■ 어디에 있나요?
  "편집" 탭 → 상단의 [📤 내보내기] 버튼 클릭 → [📂 SRT 자막 불러오기] 선택

■ 사용 조건:
  - '편집 준비'가 먼저 완료되어 있어야 합니다.
  - SRT를 불러오고나서 '편집준비'를 누르면 불러온 SRT는 소멸되고, 새로운 자막을 생성하게 됩니다.

■ 사용 방법:

  1. "편집" 탭으로 이동합니다
  2. 상단의 [📤 내보내기] 버튼을 클릭합니다
  3. 메뉴에서 [📂 SRT 자막 불러오기]를 선택합니다
  4. 파일 선택 창에서 SRT 파일을 선택합니다
  5. 자막이 교체되고 "기존 OO개 → 새로 OO개" 완료 메시지가 표시됩니다
  6. 오른쪽 자막 목록이 새 자막으로 갱신됩니다
  7. 이후 내보내기 mp4를 하면 불러온 srt를 적용하여 최종영상mp4가 출력됩니다.

■ 내보내기 메뉴 구성:
  [📤 내보내기] 클릭 시:
  ├─ 🎬 전체 영상 내보내기 (MP4)
  ├─ 📝 SRT 자막 내보내기    ← 현재 자막을 SRT로 저장
  ├─ ──────────────────
  └─ 📂 SRT 자막 불러오기    ← 외부 SRT를 가져오기  ★ 이 기능

■ SRT 파일이란?
  영상 자막에 쓰이는 표준 파일 형식입니다.
  메모장으로 열면 이런 형태입니다:

  ┌─────────────────────────────────┐
  │ 1                                │
  │ 00:00:01,000 --> 00:00:03,500    │
  │ 안녕하세요 여러분                 │
  │                                  │
  │ 2                                │
  │ 00:00:03,500 --> 00:00:06,000    │
  │ 오늘의 이야기를 시작합니다        │
  └─────────────────────────────────┘

■ 참고 사항:
  - 불러오면 기존 자막이 모두 새 자막으로 교체됩니다 (되돌리기 불가)
  - 자막의 폰트, 색상 등 스타일은 기존 프로젝트의 설정이 유지됩니다
     (SRT 파일에는 스타일 정보가 없기 때문)
  - 불러온 뒤 자막 텍스트나 타이밍을 편집 화면에서 추가로 수정할 수 있습니다
  - [3.음성] 탭의 파일을 통해 만든, SRT를 불러와야만 해당 음성과 자막 타이밍이 맞을 수 있습니다.

■ 이런 상황에서 유용합니다:
  - 자동 생성된 자막에 오타가 많아서 외부에서 직접 수정한 뒤 다시 넣고 싶을 때
  - 다른 자막 편집 도구에서 만든 SRT 파일을 사용하고 싶을 때
  - 번역된 자막 파일을 적용하고 싶을 때
  - Whisper 등 다른 STT 도구로 만든 자막을 가져오고 싶을 때

■ 자주 묻는 질문:

  Q: "먼저 편집 준비를 완료해주세요" 메시지가 떠요
  A: 편집 준비를 먼저 해야 합니다. "편집" 탭에서 [편집 준비] 버튼을 눌러 미리보기를 생성한 뒤 다시 시도해주세요.

  Q: "SRT 파일에서 자막을 찾을 수 없습니다" 메시지가 떠요
  A: SRT 파일 형식이 올바른지 확인해주세요. 표준 SRT 형식(번호, 타임라인, 텍스트)이어야 합니다. 파일을 메모장으로 열어서 형식을 확인해보세요.

  Q: 불러온 자막의 글자 스타일이 바뀌었어요
  A: SRT에는 스타일 정보가 없어서 기존 프로젝트의 스타일(폰트, 색상 등)이 자동 적용됩니다. 스타일을 변경하고 싶으면 편집 화면의 스타일 설정에서 수정해주세요.

  Q: 불러오기 전으로 되돌리고 싶어요
  A: 불러오기 전에 [📝 SRT 자막 내보내기]로 현재 자막을 백업해두시는 것을 추천합니다. 불러온 후에는 자동으로 되돌릴 수 없습니다.

  Q: 한글이 깨져서 나와요
  A: SRT 파일이 UTF-8 인코딩인지 확인해주세요. 메모장에서 "다른 이름으로 저장" → 인코딩을 "UTF-8"로 선택하면 됩니다.

## ElevenLabs TTS (일레븐랩스 음성 생성)

메이크렌즈는 ElevenLabs(일레븐랩스) API를 연동하여 AI 음성을 생성할 수 있습니다.
Gemini TTS와 더블어, 음성 탭에서 선택하여 사용할 수 있는 추가 TTS 옵션입니다.
ElevenLabs는 사용자가 직접 가입하여 무료/유료로 사용가능합니다.

> 중요: ElevenLabs는 무료 플랜 사용시 한달 1만 크레딧이 제공됩니다. 또한, 한국어 Pro 음성을 사용할 수 없을 수 있으며, 이 경우 "402 - 유료 플랜 필요" 오류가 발생합니다. '기본'음성으로 한국어도 인식은 가능하나, 어색할 수 있습니다.

### Gemini TTS vs ElevenLabs 차이점

메이크렌즈의 음성 탭에는 "Gemini TTS"와 "ElevenLabs TTS" 두 개의 서브탭이 있습니다.
탭을 클릭하면 자동으로 전환됩니다.

주요 차이점:

1) 장문 처리
- Gemini TTS: 글자 수가 많으면 음성이 깨지거나 속삭이는 현상 발생 가능. "500자 분할" 토글을 켜서 분할 생성 권장
- ElevenLabs: 500자 분할 기능 해당 없음. ElevenLabs는 대본 전체를 한 번에 API로 전송하여 생성함. 모델에 따라 한 번에 처리 가능한 글자 수가 다름 (설정 탭의 모델 비교표 참조)

2) 말투/스타일 조절
- Gemini TTS: 말투 스타일 5종 (기본, 명랑하게, 차분하게, 강하고 단호하게, 뉴스 리포터처럼) + 디렉팅 텍스트 직접 입력 + 음성 속도 슬라이더(0.5x~2.0x)
- ElevenLabs: 안정성(0~100%) 슬라이더와 속도(70~120%) 슬라이더로 조절

3) 자막 싱크 방식
- Gemini TTS: 음성 생성 후 Whisper로 음성을 분석하여 자막 타이밍 추출 (추가 시간 소요) / 오타 가능성 있음.
- ElevenLabs: API가 글자별 타임스탬프를 함께 반환하므로, Whisper 분석을 건너뛰고 바로 자막 싱크 가능 (더 빠름). 단, 타임스탬프 품질이 불량한 경우(드물게 발생) 자동으로 Whisper 분석으로 전환됨

4) 음성 파일 형식
- Gemini TTS: WAV 형식으로 저장 (narration.wav)
- ElevenLabs: MP3 형식으로 저장 (narration.mp3)

### ElevenLabs API 키 만들기 (필수)

ElevenLabs를 사용하려면 API 키가 필요합니다.

#### 1단계: ElevenLabs 회원가입
1. https://elevenlabs.io 에 접속합니다.
2. 구글 계정 또는 이메일로 회원가입합니다.

#### 2단계: API 키 발급
1. 로그인 후, 좌측 하단 사이드바의 "Developers" 항목을 찾습니다.
2. "API Keys" 탭을 클릭합니다.
3. "Create API Key" 버튼을 클릭합니다.
4. API 키 이름을 입력합니다. 예: "MakeLens"

#### 3단계: API 키 권한(Restrict Key) 설정 - 매우 중요!

API 키 생성 화면에 "Restrict Key" 토글이 있습니다.

가장 쉬운 방법: "Restrict Key"를 끄고(비활성화) 키를 생성하세요.
Restrict Key가 꺼져 있으면 모든 API 기능에 접근 가능한 키가 생성됩니다. 별도로 권한을 선택할 필요가 없습니다.
초보자분들은 이 방법을 추천합니다.

Restrict Key를 켜는 경우 (권한 직접 선택):
Restrict Key를 켜면, 기본적으로 모든 항목이 "No Access"(접근 불가) 상태입니다.
즉, 필요한 항목을 하나하나 직접 액세스 허용으로 변경해줘야 합니다.
메이크렌즈에서 필요한 권한은 다음과 같습니다:
- Text to Speech : 음성 생성에 필수. 이 권한이 없으면 음성을 만들 수 없습니다.
- Voices : 음성 목록 불러오기에 필수. 이 권한이 없으면 음성 선택 드롭다운이 비어있게 됩니다.
- User : 남은 크레딧(글자 수) 조회에 필요합니다.
- Models : 모델 목록을 불러오기에 필수.

위 항목들을 반드시 No Access에서 허용 상태로 변경해주세요.

> 권한이 부족하면 "API 키에 필요한 권한이 없습니다 (401)" 오류가 발생합니다. 이 오류가 뜨면 기존 키를 삭제하고, Restrict Key를 끈 상태로 새 키를 발급받으세요.

5. "Create" 버튼을 클릭합니다.
6. 생성된 API 키를 반드시 복사해서 저장하세요. 이 화면을 벗어나면 다시 볼 수 없습니다.
   (분실 시 새 키를 다시 발급받으면 됩니다)

> 주의: API 키는 비밀번호와 같습니다. 다른 사람에게 공유하면 그 사람이 내 크레딧을 사용할 수 있으니 절대 공유하지 마세요.
종종 'Copy Key ID'와 'API 키'를 혼동하는 분들이 계신데, 서로 다른것이며, 앞서 말씀드린대로 API키 생성시에만 API키 복사가 가능합니다.

#### 4단계: 메이크렌즈에 API 키 등록
1. 메이크렌즈를 실행합니다.
2. 상단의 "5. 설정" 탭을 클릭합니다.
3. "API설정" 서브탭에서 오른쪽의 "ElevenLabs API 관리" 영역을 찾습니다.
4. "ElevenLabs API 키를 입력하세요" 입력란에 복사한 API 키를 붙여넣기(Ctrl+V)합니다.
5. "✓ ElevenLabs API 저장" 버튼을 클릭합니다.
6. "ElevenLabs API 키가 저장되었습니다." 메시지가 나오면 완료입니다.

> 팁: 입력란 옆의 눈 아이콘(👁)을 클릭하면 입력한 API 키를 확인하거나 숨길 수 있습니다.

### ElevenLabs TTS 사용법

#### TTS 프로바이더 전환
1. 메이크렌즈에서 "3. 음성" 탭으로 이동합니다.
2. 상단에 "Gemini TTS"와 "ElevenLabs TTS" 두 개의 서브탭이 있습니다.
3. "ElevenLabs TTS" 탭을 클릭하면 자동으로 ElevenLabs로 전환됩니다.
4. 다시 "Gemini TTS" 탭을 클릭하면 Gemini로 되돌아갑니다.

#### 음성 선택
1. ElevenLabs TTS 탭을 처음 열면 "음성을 불러와주세요"로 표시됩니다.
2. API 키가 저장되어 있으면 자동으로 음성 목록을 불러옵니다.
3. 음성 목록에서 한국어 음성은 [한] 표시와 함께 상단에 우선 정렬됩니다.
4. 기본 제공 음성은 [기본] 표시가, 그 외 Professional 음성은 (Pro), 클론 음성은 (Clone) 등으로 구분됩니다.
5. 원하는 음성을 선택합니다.
6. 음성 옆의 스피커 아이콘(🔊)을 클릭하면 해당 음성의 샘플을 미리 들을 수 있습니다. 재생 중 다시 클릭하면(⏹) 정지됩니다.

#### 모델 선택

음성 선택 옆에 "모델" 드롭다운이 있습니다. 3가지 모델 중 선택할 수 있습니다.
설정 탭의 ElevenLabs 영역 하단에 모델 비교표가 있으니 참고하세요.

1) Eleven v3 (최신, 최고품질)
   - 최신 모델, 최고 음질
   - 한 번에 최대 5,000자
   - 1,000자당 약 174원 (환율 1,447원 기준, 변동 가능)

2) 다국어 v2 (안정적, 고품질)
   - 안정적이고 검증된 모델
   - 한 번에 최대 10,000자
   - 1,000자당 약 174원

3) 터보 v2.5 (빠르고 반값)
   - 생성 속도가 빠르고 가격이 절반
   - 한 번에 최대 40,000자
   - 1,000자당 약 87원

> 참고: 가격 정보는 설정 탭의 모델 비교표에서도 확인할 수 있으며, 환율에 따라 변동됩니다.

#### 남은 크레딧 확인
- 모델 선택 옆에 "남은 크레딧"이 표시됩니다. 형식: "9,532 / 10,000자" (남은 글자수 / 전체 한도)
- 새로고침 아이콘(🔄)을 클릭하면 최신 크레딧 정보를 다시 불러옵니다. (이때, 일레븐랩스에서 적용시간이 걸려, 즉시 바뀌지 않을수 있습니다.)
- 크레딧이 전체 한도의 10% 미만이면 빨간색으로 표시되어 경고합니다.
- 음성을 생성할 때마다 크레딧이 자동으로 새로고침됩니다. (이때, 일레븐랩스에서 적용시간이 걸려, 즉시 바뀌지 않을수 있습니다.
)

#### 음성 세부 설정 (슬라이더)

ElevenLabs TTS 탭 하단에 두 개의 슬라이더가 있습니다.

1) 안정성 (Stability)
   - 범위: 0% ~ 100%
   - 기본값: 50%
   - 왼쪽(0% 쪽) = 개성/다양함: 감정 표현이 풍부하고 매번 약간씩 다른 톤
   - 오른쪽(100% 쪽) = 차분/일정함: 일관되고 안정적인 음성

2) 속도 (Speed)
   - 범위: 70% ~ 120%
   - 기본값: 100% (기본 속도)
   - 왼쪽(70%) = 느리게
   - 오른쪽(120%) = 빠르게
   - ElevenLabs의 속도는 API 자체에서 조절되며, Gemini TTS처럼 후처리하는 방식이 아닙니다.

> 참고: 유사도(Similarity Boost)와 스타일(Style) 설정은 메이크렌즈에서 최적값(유사도 1.0, 스타일 0.5)으로 자동 고정되어 있습니다. 사용자가 별도로 조절할 필요 없습니다.

#### 음성 생성하기
1. 대본이 작성된 상태에서, 음성과 모델과 세부 설정을 완료합니다.
2. "🔊 AI 음성 생성" 버튼을 클릭합니다.
3. 버튼이 "⏹ 생성중지"로 바뀌며 음성이 생성됩니다.
4. 생성이 완료되면 하단에 오디오 플레이어가 나타납니다.
5. 음성 파일은 프로젝트 폴더의 "이미지 및 오디오 > 오디오" 안에 narration.mp3로 자동 저장됩니다.
6. 자막 싱크용 타임스탬프 데이터도 같은 폴더에 elevenlabs_alignment.json으로 함께 저장됩니다.

> 참고: ElevenLabs는 500자 분할 기능의 영향을 받지 않습니다. 500자 분할 토글은 Gemini TTS 전용 기능이며, ElevenLabs 사용 시에는 대본 전체가 한 번의 API 요청으로 처리됩니다.

### ElevenLabs 문제 해결 (FAQ)

Q: "ElevenLabs API 키가 입력되지 않았습니다" 오류가 뜹니다.
A: [5. 설정] > API설정 탭에서 ElevenLabs API 키를 입력하고 저장 버튼을 눌렀는지 확인하세요.
키는 ElevenLabs 사이트의 Developers > API Keys에서 발급받을 수 있습니다.

Q: "ElevenLabs 음성이 선택되지 않았습니다" 오류가 뜹니다.
A: 음성 탭의 ElevenLabs TTS 서브탭에서 음성을 먼저 선택해야 합니다.
음성 목록이 비어있다면 🔄 버튼을 눌러 목록을 불러온 후 사용할 음성을 선택해주세요.

Q: "API 키에 필요한 권한이 없습니다 (401)" 오류가 뜹니다.
A: API 키 생성 시 권한이 제한적으로 설정되었을 수 있습니다.
ElevenLabs 사이트에서 기존 키를 삭제하고, Restrict Key를 끈 상태로 새 키를 발급받아 [5. 설정]에서 다시 입력해보세요.

Q: "API 키가 유효하지 않습니다 (401)" 오류가 뜹니다.
A: API 키가 잘못 입력되었거나 만료되었습니다.
1) [5. 설정] > API설정에서 키를 다시 확인하세요.
2) 키 복사 시 앞뒤 공백이 포함되지 않았는지 확인하세요.
3) ElevenLabs 사이트에서 새 키를 발급받아 다시 등록해보세요.

Q: "이 목소리는 무료 플랜에서 사용할 수 없습니다 (402)" 오류가 뜹니다.
A: ElevenLabs 무료 사용자는 Pro 등급 목소리를 사용할 수 없습니다.
현재 한국어 목소리는 모두 Pro 등급일 수 있습니다.
해결 방법: ElevenLabs 유료 플랜으로 업그레이드합니다.

Q: "크레딧이 모두 소진되었습니다" 또는 "크레딧 부족" 오류가 뜹니다.
A: ElevenLabs 글자 수 크레딧이 다 사용되었습니다.
1) 남은 크레딧을 확인하세요 (🔄 새로고침 클릭).
2) 무료 플랜은 매달 자동 충전됩니다. 다음 달에 다시 시도하세요.
3) 또는 ElevenLabs 사이트에서 유료 플랜으로 업그레이드하세요.

Q: "요청 한도를 초과했습니다 (429)" 오류가 뜹니다.
A: 짧은 시간에 너무 많은 요청을 보냈습니다.
잠시 기다린 후 다시 시도해주세요.
메이크렌즈는 429 오류 시 자동으로 최대 3회까지 재시도합니다.
(주의 : 여기서 분당 할당만료는 1분당이 아닌, N분당입니다. 즉 1분후 될수도, 5분후,15분후 될수도 있습니다.)

Q: "요청 시간 초과 (타임아웃)" 오류가 발생합니다.
A: 대본이 너무 길거나 인터넷 연결이 불안정할 때 발생합니다.
1) 인터넷 연결 상태를 확인하세요.
2) 대본이 모델의 최대 글자 수를 초과하지 않는지 확인하세요 (v3: 5,000자, 다국어 v2: 10,000자, 터보 v2.5: 40,000자).
3) 잠시 후 다시 시도하세요.

Q: "대본이 너무 깁니다 (텍스트 길이 초과)" 오류가 뜹니다.
A: 대본 길이가 선택한 모델의 최대 글자 수를 초과했습니다.
대본을 줄이거나, 더 긴 텍스트를 지원하는 모델로 변경하세요.
- v3: 최대 5,000자
- 다국어 v2: 최대 10,000자
- 터보 v2.5: 최대 40,000자

Q: "선택한 음성을 찾을 수 없습니다" 오류가 뜹니다.
A: 해당 음성이 ElevenLabs에서 삭제되었거나 비공개 상태일 수 있습니다.
🔄 버튼을 눌러 음성 목록을 새로고침한 후 다른 음성을 선택해주세요.

Q: 인터넷 연결 오류가 발생합니다.
A: ElevenLabs 서버에 접속할 수 없는 상태입니다.
1) 인터넷 연결 상태를 확인해주세요.
2) Wi-Fi 또는 유선 연결이 정상인지 확인해주세요.
3) VPN을 사용 중이라면 잠시 끄고 다시 시도해보세요.

Q: ElevenLabs로 생성한 음성의 자막 싱크가 안 맞습니다.
A: 드물게 ElevenLabs의 타임스탬프 데이터가 부정확할 수 있습니다.
이 경우 메이크렌즈가 자동으로 감지하여 Whisper 분석으로 전환합니다.
만약 그래도 자막이 안 맞는다면, 음성을 다시 한번 생성해보세요.

Q: Gemini TTS에서 ElevenLabs로 바꿨는데, 기존 음성은 어떻게 되나요?
A: TTS 탭을 전환하는 것만으로는 기존 음성 파일이 삭제되지 않습니다.
"🔊 AI 음성 생성" 버튼을 눌러야만 새로운 TTS 엔진으로 음성이 재생성됩니다.
새로 생성하면 기존 파일(Gemini의 narration.wav)은 자동 정리되고, ElevenLabs의 narration.mp3로 대체됩니다.

Q: ElevenLabs의 음성을 미리 들어볼 수 있나요?
A: 네. 음성 선택 드롭다운 옆의 스피커 아이콘(🔊)을 클릭하면 해당 음성의 샘플을 들을 수 있습니다.
모든 음성에 샘플이 있는 것은 아니며, 샘플이 없으면 "미리듣기 샘플이 없습니다" 안내가 표시됩니다.

Q: 음성 목록을 새로고침하려면 어떻게 하나요?
A: ElevenLabs TTS 탭으로 전환하면 음성 목록이 비어있을 경우 자동으로 불러옵니다.
수동으로 새로고침하려면 음성 선택 영역의 🔄 버튼을 클릭하면 됩니다.

Q: 남은 크레딧 숫자가 갱신이 안 됩니다.
A: 크레딧 옆의 🔄 새로고침 버튼을 클릭해주세요.
음성 생성 후에는 자동으로 크레딧이 갱신되지만, 약간의 시간차가 있을 수 있습니다.

### ElevenLabs 사용 팁

- 한국어 음성은 목록 상단에 [한] 표시와 함께 자동 정렬됩니다.
- 안정성 슬라이더를 50~80%로 설정하면 나레이션에 적합한 일관된 음성을 얻기 좋습니다.
- 터보 v2.5 모델은 가격이 절반이고 최대 40,000자까지 지원하므로, 대량 작업이나 비용 절감에 유리합니다.
- 음성이 마음에 들지 않으면, 안정성과 속도 슬라이더를 조절한 뒤 다시 생성해보세요. 단, 다시 생성할 때마다 크레딧이 차감됩니다.
- 오류 발생 시, 에러 대화상자의 "상세 에러 보기"를 열어 기술 지원에 전달하면 정확한 원인 파악에 도움이 됩니다.

### 캐릭터 시스템 업데이트

#### 변경 요약
기존에는 캐릭터를 최대 3개(char_1 ~ char_3)까지만 등록할 수 있었습니다.
이번 업데이트로 최대 7개(char_1 ~ char_7)까지 등록 가능하며, 캐릭터 참조 이미지가 이미지 생성에 정확히 반영되도록 개선되었습니다.

주요 변경:
- 캐릭터 슬롯 3개 → 7개 확장
- 펼침/접기 UI 추가 (char_4 ~ char_7)
- 캐릭터 참조 이미지가 이미지 생성에 정확히 반영되도록 수정
- 다중 캐릭터(2명 이상)를 한 장면에 사용할 때 각 캐릭터가 올바르게 매칭되도록 개선
- 딸깍(일괄생성) 다이얼로그도 7개 지원


#### 캐릭터 패널 UI
장면 탭 상단의 캐릭터 영역이 변경되었습니다.

1) 메인 영역 (항상 표시)
   - char_1, char_2, char_3 버튼 3개가 기존과 동일하게 표시됩니다.
   - 우측에 펼침 토글 버튼이 새로 추가되었습니다.

2) 토글 버튼
   - 접힌 상태: "+4" 라고 표시되며, 점선 테두리로 되어 있습니다.
   - 펼친 상태: "▲" 로 바뀌며, 파란색 테두리로 변합니다.
   - 클릭하면 아래에 추가 캐릭터 패널이 나타납니다.

3) 펼침 패널
   - char_4, char_5, char_6, char_7 버튼 4개가 고정 배치되어 있습니다.
   - 장면 목록 위에 레이어(플로팅)로 표시되므로, 장면을 아래로 밀지 않습니다.
   - 캐릭터 영역 아래 우측에 정렬됩니다.

> 참고: 펼침 패널은 토글 버튼을 다시 클릭하면 접힙니다. 다른 영역을 클릭해도 자동으로 닫히지 않으므로, 토글 버튼으로 직접 닫아야 합니다.


#### 캐릭터 등록/삭제
1. 등록하려는 캐릭터 버튼(char_1 ~ char_7)을 좌클릭합니다.
2. 파일 선택 창이 열리면 원하는 캐릭터 이미지를 선택합니다.
3. 버튼에 선택한 이미지의 썸네일이 표시되면 등록 완료입니다.
4. 등록된 캐릭터를 삭제하려면 해당 버튼을 우클릭합니다.
5. 버튼 위에 마우스를 올리면 등록된 이미지의 미리보기가 나타납니다.

> 참고: 등록된 캐릭터 이미지는 프로젝트 저장 시 함께 저장되며, 프로젝트를 다시 열면 자동으로 복원됩니다.


#### 프롬프트에서 캐릭터 사용하기
이미지 프롬프트에 char_1 ~ char_7 을 텍스트로 직접 입력하면, 해당 캐릭터의 참조 이미지가 이미지 생성 시 자동으로 적용됩니다.

사용 예시:
- 단일 캐릭터: "와이드 샷. char_1이 카페에서 커피를 마시고 있다."
  → char_1에 등록된 캐릭터 이미지를 참조하여 생성됩니다.

- 다중 캐릭터: "char_4와 char_5가 함께 거리를 걷고 있다."
  → char_4와 char_5에 등록된 캐릭터 이미지가 모두 적용됩니다.

- 캐릭터 교체: 기존 프롬프트에서 "char_1"을 "char_5"로 수동 수정
  → 수정된 대로 char_5의 참조 이미지만 적용됩니다. (기존에 분석된 char_1은 무시됨)

> 참고: 프롬프트에 char_X 텍스트가 있어야만 해당 캐릭터가 참조됩니다. 해당 번호에 이미지가 등록되어 있지 않으면 참조 없이 생성됩니다.


#### 딸깍(일괄생성) 다이얼로그
딸깍 다이얼로그의 캐릭터 영역도 동일하게 7개로 확장되었습니다.
- 7가지 색상으로 구분됩니다: 빨강, 파랑, 초록, 노랑, 보라, 청록, 주황
- 등록/삭제 방법은 메인 화면과 동일합니다. (좌클릭 = 등록, 우클릭 = 삭제)


#### 기존 사용자 참고사항 (기존 대비 달라진 동작)

1) 프롬프트 수동 편집이 정확히 반영됩니다.
   - 기존: 장면 분석 시 등장캐릭터로 지정된 char_1이, 사용자가 프롬프트에서 char_5로 바꿔도 char_1까지 같이 참조되는 문제가 있었습니다.
   - 변경: 프롬프트에 직접 쓴 캐릭터만 참조됩니다. 프롬프트에 char_X가 하나도 없을 때만 장면 분석 결과를 fallback으로 사용합니다.
2) 캐릭터 설명이 없어도 참조 이미지가 정상 작동합니다.
3) Whisk에서 참조 이미지가 확실히 반영됩니다.

### 캐릭터 문제 해결 (FAQ)

Q: 캐릭터 이미지를 등록했는데 생성된 이미지에 반영이 안 됩니다.
A: 이미지 프롬프트에 char_1, char_2 등 캐릭터 ID가 텍스트로 포함되어 있는지 확인하세요.
프롬프트에 char_X가 없으면 참조 이미지가 전달되지 않습니다.

Q: 프롬프트에서 char_1을 char_5로 바꿨는데 여전히 char_1 캐릭터가 나옵니다.
A: 이전 버전의 문제입니다. 업데이트 후에는 프롬프트에 쓴 캐릭터만 정확히 참조됩니다.
프롬프트 텍스트를 다시 확인하고 이미지를 재생성해보세요.

Q: 캐릭터를 2명 넣었는데 둘 다 반영이 안 됩니다.
A: 다중 캐릭터는 각각의 캐릭터 슬롯에 이미지가 등록되어 있어야 합니다.
예를 들어 프롬프트에 "char_4와 char_5"를 쓰면, char_4와 char_5 모두에 이미지가 등록되어 있어야 합니다.

Q: char_4 ~ char_7 버튼이 보이지 않습니다.
A: 캐릭터 영역 우측의 "+4" 토글 버튼을 클릭하면 char_4 ~ char_7 패널이 아래에 펼쳐집니다.

Q: Whisk에서 캐릭터 1명은 잘 되는데 2명은 느리거나 품질이 다릅니다.
A: Whisk는 결과물의 스타일이나 품질이 달라질 수 있으며, 이는 Whisk 자체의 특성입니다.

## 가장 중요한 추가: `장면 재실행` 버튼

- 실패한 장면을 전체 재시작 없이, 확장앱에서 해당 장면만 바로 다시 실행할 수 있습니다.
- 즉, 예전처럼 전체 큐를 다시 돌릴 필요 없이 "문제 난 장면만" 빠르게 복구할 수 있습니다.

사용 방법:
1. Grok 작업 중 실패 장면이 생기면, 해당 장면의 `재실행` 버튼을 누릅니다.
2. 메이크렌즈가 그 장면 데이터(프롬프트/이미지)를 다시 요청해 대기열에 재등록합니다.
3. 재등록된 장면만 다시 생성됩니다.

이 기능이 특히 유용한 경우:
- 특정 장면만 일시적 서버 오류(500/503)로 실패한 경우
- 다운로드 타이밍 이슈로 일부 장면만 누락된 경우
- 긴 배치 작업에서 1~2개 장면만 실패한 경우

## 무엇이 좋아졌나요?

1. 일시적 오류 자동 재시도 강화
- 이전보다 다양한 오류(서버 일시 장애, 네트워크 불안정, 페이지 연결 끊김)를 구분해서 자동 재시도합니다.
- 재시도 대기시간도 오류 종류에 맞게 다르게 적용됩니다.

2. 오류 메시지 안내 개선
- 기술적인 문구 대신 사용자 기준 안내 문구로 표시됩니다.
- 예: "서버 불안정", "네트워크 불안정", "연결 끊김", "요청 제한(429)" 등.

3. 연결 끊김 복구 안정성 향상
- Grok 탭/확장앱 간 연결 이상을 더 빨리 감지하고 재연결을 시도합니다.
- 작업 중 끊겨도 상태 유실을 줄여서 복구 가능성을 높였습니다.

4. 다운로드 누락 보완
- 다운로드 버튼 클릭이 누락된 경우 자동 재시도 루틴이 동작합니다.
- 완료 신호 중복 처리(같은 장면 중복 완료 표시)도 더 강하게 방지합니다.

# v1.5.5 업데이트
## 번역 기능

메이크렌즈는 작성한 대본을 다국어로 번역하여 해외 채널 운영이나 다국어 영상 제작에 활용할 수 있는 번역 기능을 제공합니다. 
AI를 활용한 자연스러운 번역으로, 단순 직역이 아닌 해당 언어의 유튜브 문화에 맞는 네이티브 스타일 번역을 제공합니다.

### 지원 언어

현재 다음 5개 언어를 지원합니다:

| 언어 | 코드 | 아이콘 |
|------|------|--------|
| 영어 | en | 🇺🇸 |
| 일본어 | ja | 🇯🇵 |
| 스페인어 | es | 🇪🇸 |
| 포르투갈어 (브라질) | pt | 🇧🇷 |
| 한국어 | ko | 🇰🇷 |
*한국어 번역  기능은, 원본대본을 타 국가 대본을 넣었을때 사용하기 유용합니다*

### 사전 준비

번역 기능을 사용하기 위해서는 다음 조건이 충족되어야 합니다:

1. API 키 설정: 메이크렌즈의 API 키가 설정되어 있어야 합니다. API 키가 없으면 "API 키가 설정되지 않았습니다." 알림이 표시됩니다.
2. 대본 작성 완료: 번역할 대본이 존재해야 합니다. 대본이 비어있으면 "번역할 대본이 없습니다." 알림이 표시됩니다.

### 번역 실행 방법

#### 1단계: 번역 컨트롤 확인

[1. 대본] 탭의 "유튜브용 대본" 영역 우측 상단에 번역 컨트롤 그룹이 있습니다.

- 언어 선택 콤보박스: 🇺🇸 영어, 🇯🇵 일본어, 🇪🇸 스페인어, 🇧🇷 포르투갈어, 🇰🇷 한국어 중 선택
- 번역 버튼: 주황색 텍스트로 표시된 "번역" 버튼
- ON/OFF 토글 버튼: 번역 활성/비활성 전환 버튼

#### 2단계: 언어 선택 및 번역 실행

1. 콤보박스에서 원하는 번역 언어를 선택합니다.
2. "번역" 버튼을 클릭합니다.
3. 번역이 시작되면 버튼이 "⏹ 중지" (빨간색)로 바뀌며 진행 상태가 표시됩니다.
4. 번역이 완료되면 자동으로 번역 ON 상태로 전환됩니다.

#### 3단계: 번역 결과 확인

번역이 완료되면:
- 토글 버튼이 "ON 🇺🇸" (해당 언어 국기) 형태의 초록색 버튼으로 바뀝니다.
- 원본 대본 오른쪽에 번역 대본 패널이 표시됩니다 (초록색 텍스트).

### 번역 ON/OFF 토글

번역 기능의 핵심은 ON/OFF 토글입니다. 이 토글은 번역된 대본을 영상 제작 과정 전체에 적용할지 결정합니다.

**ON 상태 (초록색 버튼)**:
- 토글 버튼에 "ON" + 해당 언어 국기 아이콘이 표시됩니다.
- 원본 대본 옆에 번역 대본 패널이 나타납니다.
- [2. 장면/이미지] 탭의 각 장면 카드 아래에 번역 텍스트가 표시됩니다.
- [3. 음성] 탭의 대본에 번역 텍스트가 반영됩니다.
- [4. 편집] 탭에서 편집준비 실행 시 번역 대본이 자막으로 적용됩니다.

**OFF 상태 (회색 버튼)**:
- 토글 버튼에 "OFF"가 회색으로 표시됩니다.
- 번역 대본 패널이 숨겨집니다.
- 모든 탭에서 원본 한국어 대본이 사용됩니다.

**토글 전환 조건**:
- ON으로 전환하려면 번역이 이미 존재해야 합니다.
- OFF로 전환하면 번역 데이터가 삭제되지는 않으며, 언제든 다시 ON으로 전환할 수 있습니다.


### 번역 중지

번역 진행 중 "⏹ 중지" 버튼을 클릭하면 번역이 즉시 중단됩니다. 중단된 번역은 저장되지 않으며, 다시 번역 버튼을 눌러 처음부터 시작해야 합니다.


### 이미 번역이 있는 경우 재번역

이미 번역이 존재하는 언어에 대해 다시 번역 버튼을 누르면 확인 대화상자가 나타납니다:
"이미 [언어] 번역이 존재합니다. 다시 번역하시겠습니까?"
"예"를 선택하면 기존 번역을 덮어쓰고 새로 번역합니다.


### 장면/이미지 탭에서의 번역 표시 및 수정

#### 번역 바

[2. 장면/이미지] 탭 상단에 별도의 번역 컨트롤 바가 있습니다. 이 바에는 언어 선택 콤보박스와 ON/OFF 토글 버튼이 포함되어 있습니다. 대본 탭의 번역 상태와 동기화되어 동작합니다. 이 번역 바는 스크롤해도 상단에 고정되어 항상 보입니다.

#### 장면별 번역 텍스트 표시

번역 ON 상태일 때, 각 장면 카드 하단에 번역된 텍스트가 초록색으로 표시됩니다. 국기 아이콘과 언어 코드(예: 🇺🇸 EN)가 라벨로 함께 표시됩니다.

#### 장면별 번역 수정

각 장면 카드의 번역 텍스트 영역은 편집 가능합니다. 직접 수정하면:
- 해당 장면의 번역 데이터(translated_script)가 즉시 업데이트됩니다.
- 전체 번역 대본(full_script)도 자동으로 재조합됩니다.
- 음성 탭에서 사용하는 활성 대본에도 즉시 반영됩니다.


### 음성 탭에서의 번역 연동

[3. 음성] 탭에도 번역 바가 있으며, 대본 탭/장면 탭의 번역 상태와 동기화됩니다. 번역 ON 상태에서 음성을 생성하면 번역된 대본 기준으로 음성이 만들어집니다.


### 편집준비에서의 번역 대본 적용

[4. 편집] 탭에서 "편집준비"를 실행할 때, 번역이 ON 상태이면 다음과 같이 동작합니다:

1. **대본 교체**: 원본 한국어 대본 대신 번역된 대본이 각 장면에 적용됩니다. 장면 번호(No)를 기준으로 매칭하여 대본 열의 텍스트를 번역 텍스트로 교체합니다.
2. **자막 폰트 자동 전환**: 번역 대본 적용 시 자막 폰트가 자동으로 "Pretendard"로 전환됩니다 (다국어 문자 지원을 위함). 사용자가 이후 수동으로 폰트를 변경하면 변경한 폰트가 유지됩니다.
3. **Whisper 언어 힌트**: 활성 번역 언어가 음성 인식(Whisper)의 언어 힌트로 전달되어, 번역 언어에 맞는 정확한 음성-텍스트 매칭이 이루어집니다.
4. **나레이션 파일 탐색**: 번역 언어에 해당하는 나레이션 파일(예: narration_en.mp3, narration_ja.wav)을 우선적으로 탐색합니다. 해당 파일이 없으면 기본 narration.wav를 사용합니다.


### 번역 저장 및 로드

#### 자동 저장

번역 데이터는 프로젝트 단위로 자동 저장됩니다:
- 번역 완료 시: 번역 결과(translations)와 활성 번역 언어(active_translation_lang)가 project.json에 저장됩니다.
- 번역 ON/OFF 토글 변경 시: 활성 언어 상태가 project.json에 저장됩니다.
- 번역 언어 변경 시: 변경된 활성 언어가 project.json에 저장됩니다.

#### 프로젝트 로드 시 복원

프로젝트를 다시 열면:
- 저장된 번역 데이터가 자동으로 복원됩니다.
- 마지막으로 활성화된 번역 언어가 콤보박스에 설정됩니다.
- ON/OFF 토글 상태가 복원됩니다.
- 번역 대본 표시 영역에 해당 언어의 번역 텍스트가 표시됩니다.

#### 다중 언어 저장

여러 언어로 번역한 경우, 모든 번역 결과가 언어별로 저장됩니다. 콤보박스에서 언어를 전환하면 해당 언어의 기존 번역이 있으면 즉시 표시됩니다.


### 번역 내부 동작 방식

번역은 대본 전체를 하나의 단위로 번역합니다. 장면 분석 여부와 관계없이 동작합니다.

#### 번역 처리 방식

1. **전체 대본 번역**: 대본 전체 텍스트를 Gemini에 전달하여 한 번에 번역합니다. 줄 단위(1:1)로 원본과 번역이 대응되도록 번역됩니다.

2. **장면별 자동 매칭**: 장면 분석이 완료된 상태에서 번역이 존재하면, 번역을 장면별로 자동 매칭합니다.
   - 원본 대본의 각 줄과 번역된 줄을 순서대로 매칭하여 즉시 완료됩니다.
   - 장면 분석 후 자동 실행되며, 번역 ON 상태에서 장면 탭 진입 시에도 자동 실행됩니다.
   - 대본 길이에 관계없이 문제없이 동작합니다.

#### 번역 스타일

Gemini AI에 각 언어별 유튜브 크리에이터 스타일 가이드라인이 전달됩니다:
- **영어**: 캐주얼한 유튜브 영어 ("insane", "no way", "bruh" 등)
- **일본어**: ため口 또는 です/ます, ヤバい, マジで 등 자연스러운 표현
- **스페인어**: 라틴 아메리카 스페인어 유튜브 스타일
- **포르투갈어**: 브라질 포르투갈어 유튜브 스타일
- **한국어**: 자연스러운 구어체 한국어


### 에러 발생 시 대처법

번역 중 발생할 수 있는 오류와 해결 방법:

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|-----------|
| API 키가 설정되지 않았습니다. | API 키 미설정 | 설정에서 Vertex AI API Json 파일을 등록하세요. |
| 번역할 대본이 없습니다. | 대본이 비어있음 | 대본을 먼저 작성하세요. |
| API 키가 유효하지 않습니다. | API 키 오류 또는 만료 | 설정에서 올바른 API 키를 확인하세요. |
| API 요청 한도를 초과했습니다. | 사용량 초과 | 1~2분 후 번역 버튼을 다시 눌러주세요. |
| 서버가 일시적으로 응답하지 않습니다. | 서버 과부하 | 잠시 후 번역 버튼을 다시 눌러주세요. |
| 요청 시간이 초과되었습니다. | 네트워크 타임아웃 | 네트워크 연결 확인 후 다시 시도하세요. |
| 번역 중 오류가 발생했습니다. | 기타 오류 | 번역 버튼을 다시 눌러주세요. |
| 번역에 실패했습니다. | 3회 재시도 후에도 실패 | 서버 상태가 불안정할 수 있으니 잠시 후 다시 시도하세요. |

참고: 서버 오류나 요청 한도 초과 시 자동으로 최대 3회까지 재시도합니다. 재시도 시 대기 시간이 점진적으로 증가하며(3초, 6초), 서버 과부하 시에는 자동으로 안정적인 대체 모델로 전환하여 재시도합니다.


### 주의사항

1. **대본 수정 시 재번역 필요**: 원본 대본을 수정한 후에는 번역이 자동 갱신되지 않습니다. 대본을 수정했다면 번역을 다시 실행해야 합니다.
2. **프로젝트 저장 필요**: 번역 데이터는 프로젝트 단위로 저장되므로, 프로젝트가 생성/저장된 상태에서 번역해야 데이터가 유지됩니다.
3. **번역 ON 상태에서 편집준비**: 편집준비를 실행하기 전에 번역 ON/OFF 상태를 확인하세요. ON 상태이면 번역 대본이 자막에 적용되고, OFF 상태이면 원본 대본이 적용됩니다.
4. **자막 폰트**: 번역 ON 상태에서 편집준비 시 자막 폰트가 Pretendard로 자동 변경됩니다. 다른 폰트를 원하면 편집준비 후 수동으로 변경하세요.
5. **장면 분석과 번역 순서 무관**: 번역과 장면 분석은 어느 것을 먼저 하든 상관없습니다. 나중에 장면 분석을 실행하면 번역이 자동으로 장면별 매칭됩니다.

  Q: Gemini TTS와 일레븐랩스 중 어떤 게 자막이 더 정확한가요?
  A: 일레븐랩스가 글자 단위 타이밍을 제공하므로 자막 싱크가 더 정확합니다.
     Gemini TTS는 음성 인식 기반이라 약간의 오차가 있을 수 있습니다.

  Q: 자막 줄바꿈이 어색하게 되는 경우가 있어요
  A: 글자수 제한 설정을 조정하거나, 대본의 문장 구조를 수정해보세요.
     자연스러운 문장 끝에서 줄바꿈되도록 개선되었지만, 대본 구조에 따라 다를 수 있습니다.

  - 인터넷 연결이 필요합니다

■ 사용 방법:
  1. "설정" 탭으로 이동합니다

  [무료 API인 경우 또는 결제계정연동이나 vertex AI API 활성화를 안한 경우 - 실패]

  - "지금 등록한 키는 무료 API일 가능성이 높습니다" 메시지가 표시됩니다
  - 크레딧 '프로젝트'와 API의 '프로젝트'를 다시 확인해달라는 안내가 나옵니다

■ 참고 사항:
  - 테스트는 나노바나나 일반(gemini-2.5-flash-image) 모델로 이미지 1장을 생성합니다

■ 자주 묻는 질문:
  Q: 유료 API인데 실패가 떠요
  A: - 일시적인 서버 과부하일 수 있습니다. 잠시 후 다시 시도해보세요.
     - Vertex AI API 활성화가 안되었거나, 결제계정연동을 안했을 가능성이 높습니다. 영상을 다시 확인하시고 1~5단계까지 모두 완료하세요. (https://youtu.be/tbU1uNAQL0Q)


6. 프롬프트 개선 (이미지/동영상 프롬프트 개선 및 개별 재생성)
■ 한줄 설명:
  이미지와 동영상 프롬프트가 더 정확하게 생성되도록 개선되었고,

■ 개선된 점:

  [이미지 프롬프트 개별 재생성]

  1. 장면 카드에서 동영상 프롬프트 옆의 초록색 ✹ 버튼을 클릭합니다
  2. AI가 해당 장면의 대본을 다시 분석하여 동영상 프롬프트만 새로 만듭니다
  3. 이미지 프롬프트와 다른 장면은 그대로 유지됩니다

■ 버튼 정리:
  - ✹ (노란색) : 이미지 프롬프트 개별 재생성
  - ✹ (초록색) : 동영상 프롬프트 개별 재생성

## 썸네일 기능

메이크렌즈는 AI를 활용하여 유튜브 썸네일을 자동으로 생성하는 기능을 제공합니다.
텍스트 문구 입력, 벤치마크 이미지 참고, 대본 기반 자동 문구 추출 등 다양한 방식으로 클릭을 유도하는 매력적인 썸네일을 한 번에 4장까지 만들 수 있습니다.

### 지원 화면 비율

- 16:9 (롱폼): 일반 유튜브 영상용 가로형
- 9:16 (쇼츠): 유튜브 쇼츠용 세로형

*쇼츠(9:16) 모드에서는 생성된 썸네일을 쇼츠 영상의 첫 프레임으로 고정하는 📌 핀 기능을 추가로 사용할 수 있습니다.*

### 생성 모델 선택

썸네일 탭 상단에서 생성 모델을 선택할 수 있습니다.

**나노2 (API)**:
- Google Gemini의 나노 바나나2 이미지 생성 모델을 사용합니다.
- API 키가 필요합니다.
- 한글 텍스트 렌더링이 가능합니다.
- 벤치마크 이미지의 스타일(구도, 색감, 분위기)을 직접 참고하여 비슷한 느낌으로 생성합니다.
- 4장을 동시에 병렬 생성하여 빠릅니다.

**Flow (확장앱)**:
- Google Flow 서비스를 Chrome 확장앱을 통해 사용합니다.
- API 키 없이도 사용할 수 있습니다.
- 참조 이미지(벤치마크, 추가 이미지)를 Flow에 전달하여 참고합니다.
- 한 장씩 순차적으로 생성됩니다 (Flow 특성상 동시 생성 불가).
- 사용하려면 Chrome 확장앱(메이크렌즈 확장앱)이 설치되어 있어야 합니다.
- 처음 사용 시 Chrome이 자동으로 실행되고 확장앱 연결을 대기합니다.
- 크롬창은 확장앱이 열려있는 1개만 띄워주세요.
- 확장앱을 켜서 '연결됨'을 확인해주세요.
- 썸네일 이미지생성이 크롬 백그라운드에서 생성되어 자동저장됩니다.

**나노2와 Flow의 차이점** (둘 다 내부적으로 같은 나노2 모델을 사용하지만, 접근 방식이 다릅니다):
- 한글 텍스트: 둘 다 한글 렌더링이 가능합니다. 다만 나노2가 더 정확한 편입니다.
- 벤치마크 스타일 반영: 나노2는 벤치마크 이미지의 구도/색감/분위기를 직접 참고합니다. Flow는 참조 이미지로 전달되지만 스타일 반영이 덜 정확할 수 있습니다.
- 생성 속도: 나노2는 4장을 동시에 병렬 생성합니다. Flow는 한 장씩 순차 생성이라 상대적으로 느립니다.
- API 키: 나노2는 API 키가 필요합니다. Flow는 API 키 없이 무료로 사용 가능합니다.
- 수정 재생성: 나노2는 기존 이미지를 기반으로 수정합니다. Flow는 수정 지시를 프롬프트에 추가하여 새로 생성합니다.
- 생성 중 크롬창: Flow 사용 중에는 크롬창을 닫지 마세요. 닫으면 생성이 실패합니다.

**어떤 모델을 선택해야 하나요?**
- 버텍스 API 키가 있고 한글 텍스트가 포함된 썸네일이 필요하다면 → 나노2 (API) 추천
- API 키 없이 사용하고 싶거나, 나노2로 생성이 안 될 때 → Flow (확장앱) 추천
- 두 모델로 각각 생성해보고 마음에 드는 쪽을 선택하는 것도 좋은 방법입니다.

### 사전 준비

**나노2 (API) 모드**:
1. API 키 설정: 메이크렌즈의 API 키가 설정되어 있어야 합니다. API 키가 없으면 "API 키가 설정되지 않았습니다." 알림이 표시됩니다.
2. 프로젝트 생성: 썸네일이 저장될 프로젝트가 생성되어 있어야 합니다.

**Flow (확장앱) 모드**:
1. Chrome 확장앱 설치: 메이크렌즈 Chrome 확장앱이 설치되어 있어야 합니다.
2. 프로젝트 생성: 썸네일이 저장될 프로젝트가 생성되어 있어야 합니다.
3. Flow 선택 후 생성 버튼을 누르면 Chrome이 자동으로 실행되고, 확장앱 연결을 기다립니다 (최대 30초).

### 썸네일 탭 화면 구성

썸네일 탭은 좌우 2분할 레이아웃으로 구성됩니다.

**왼쪽 패널 (설정 영역)**:
- 📝 텍스트 입력 카드: 썸네일에 들어갈 문구 설정 (모델 선택, 화면 비율, 대본 추출 포함)
- 📎 참고 이미지 카드: 벤치마크 및 추가 이미지 업로드
- 🎨 썸네일 생성 버튼

**오른쪽 패널 (결과 영역)**:
- 🖼️ 생성 결과: 2×2 격자로 4장의 썸네일 표시
- 각 셀에 수정/재생성/핀 버튼 포함

### 썸네일 생성 방법

#### 1단계: 모델 및 비율 선택

📝 텍스트 카드 상단에서 생성 모델과 화면 비율을 선택합니다.

**생성 모델 선택**:
- "나노2 (API)" 또는 "Flow (확장앱)" 중 선택합니다.

**화면 비율 선택**:
- "롱폼 16:9" 또는 "쇼츠 9:16"을 선택합니다.
- 영상 유형에 따라 자동으로 기본값이 설정됩니다.

#### 2단계: 텍스트 문구 입력

📝 텍스트 카드에서 썸네일에 표시할 문구를 설정합니다.

**문구 입력 방식 (2가지)**:

방법 1 - 직접 입력:
- 4개의 텍스트 입력란에 원하는 문구를 직접 작성합니다.
- 줄바꿈을 사용하여 여러 줄로 나눌 수 있습니다. (엔터키로 줄바꿈)
- 한 줄에 10자 이내를 권장합니다 (가독성을 위해).

방법 2 - AI 자동 추출:
- "🔍 대본에서 추출" 버튼을 클릭합니다.
- AI가 대본을 분석하여 클릭을 유도하는 4개의 문구를 자동 생성합니다.
- 호기심 유발, 숫자 활용, FOMO(놓칠까봐 두려운 심리), 권위/사회적 증거, 감정 자극, 반전/대비, 도발 등 다양한 심리 기법이 활용됩니다.
- 추출된 문구는 텍스트 입력란에 자동으로 채워지며, 필요에 따라 수정할 수 있습니다.

**문구 선택/해제**:
- 각 입력란의 체크박스로 개별 썸네일의 생성 여부를 선택/해제할 수 있습니다.
- "전체 해제/선택" 버튼으로 한 번에 전체를 토글할 수 있습니다.
- 문구를 비우고 체크한 상태로 생성하면, 텍스트 없이 이미지만으로 구성된 썸네일이 만들어집니다.

#### 3단계: 참고 이미지 업로드 (선택사항)

📎 참고 이미지 카드에서 스타일 참고 자료를 업로드할 수 있습니다. 이 단계는 선택사항이며, 참고 이미지 없이도 썸네일 생성이 가능합니다.

**벤치마크 썸네일 (최대 3장)**:
- 스타일을 참고할 유튜브 썸네일 이미지를 업로드합니다.
- 나노2 모드: 비슷한 분위기/구도/색감의 썸네일을 생성하는 데 직접 활용됩니다.
- Flow 모드: 참조 이미지로 전달되어 참고됩니다.
- 드래그 앤 드롭 또는 클릭으로 업로드할 수 있습니다.

**추가 이미지 (최대 3장)**:
- 썸네일에 포함시킬 인물 사진이나 제품 이미지를 업로드합니다.
- 업로드된 이미지의 인물/제품이 썸네일에 반영됩니다.
- 드래그 앤 드롭 또는 클릭으로 업로드할 수 있습니다.

*지원 파일 형식: PNG, JPG, JPEG, WebP, BMP*
*업로드된 이미지의 ✕ 버튼을 클릭하면 해당 이미지를 제거할 수 있습니다.*

**참고 이미지와 장면 이미지의 관계**:
벤치마크 이미지를 업로드하지 않은 경우, AI 문구 추출 시 연결된 장면 이미지가 자동으로 참고 자료로 활용됩니다.

#### 4단계: 썸네일 생성 실행

1. "🎨 썸네일 생성 (N장)" 버튼을 클릭합니다 (N은 선택된 썸네일 수).
2. 생성이 시작되면 버튼이 "⏹ 생성 중지" (빨간색)로 바뀌며 진행 상태가 표시됩니다.
3. 나노2 모드: 4장이 동시에 병렬 생성됩니다. / Flow 모드: 한 장씩 순차적으로 생성됩니다.
4. 각 썸네일이 완성될 때마다 오른쪽 결과 영역에 순차적으로 표시됩니다.
5. 모든 생성이 완료되면 프로젝트 폴더에 자동 저장됩니다.

**Flow 모드 첫 사용 시**:
- Flow를 처음 사용하면 Chrome 브라우저가 자동으로 실행됩니다.
- 확장앱이 연결될 때까지 최대 30초 대기합니다.
- 연결이 완료되면 자동으로 썸네일 생성이 시작됩니다.
- 이미 장면/이미지 탭에서 Flow를 사용한 적이 있다면 별도 연결 없이 바로 생성됩니다.

### 생성 결과 확인 및 활용

생성된 4장의 썸네일은 오른쪽 패널의 2×2 격자에 표시됩니다.

**각 썸네일 셀 구성**:
- 이미지 미리보기: 클릭하면 전체 크기로 확인 가능
- #1~#4 번호 라벨
- 수정 지시 입력란: 수정 사항을 텍스트로 입력
- 🔄 재생성 버튼: 해당 썸네일만 다시 생성
- 📌 핀 버튼: 쇼츠 첫 프레임 설정 (9:16 모드에서만 표시)

### 개별 썸네일 수정 및 재생성

생성된 썸네일이 마음에 들지 않으면 개별적으로 수정하거나 재생성할 수 있습니다.
재생성 시에도 현재 선택된 생성 모델(나노2 또는 Flow)이 사용됩니다.

**수정 재생성 (수정 지시 입력 후 🔄 클릭)**:
- 수정 지시 입력란에 원하는 변경사항을 적습니다.
- 예: "배경을 더 어둡게", "빨간색 추가", "인물을 더 크게"
- 🔄 버튼을 클릭하면 기존 이미지를 기반으로 수정된 버전이 생성됩니다.
- Flow 모드에서는 수정 지시가 프롬프트에 추가되어 새로 생성됩니다.

**새로 재생성 (수정 지시 없이 🔄 클릭)**:
- 수정 지시 입력란을 비운 채 🔄 버튼을 클릭합니다.
- 동일한 설정으로 완전히 새로운 썸네일이 생성됩니다.

### 쇼츠 첫 프레임 핀 기능

9:16 (쇼츠) 모드에서만 사용할 수 있는 특별 기능입니다.

**📌 핀 버튼 사용법**:
1. 화면 비율이 "쇼츠 9:16"으로 설정되어 있어야 합니다.
2. 원하는 썸네일의 📌 버튼을 클릭합니다.
3. 선택된 셀의 📌 버튼이 주황색으로 강조됩니다.
4. 해당 썸네일이 쇼츠 영상의 첫 프레임으로 설정됩니다.

**핀 기능 특징**:
- 한 번에 1장만 핀으로 지정할 수 있습니다 (다른 셀을 핀하면 기존 핀이 해제됩니다).
- 이미 핀된 셀을 다시 클릭하면 핀이 해제됩니다.
- 핀 상태는 [4. 편집] 탭에 자동으로 전달됩니다.
- 편집 탭에서 영상 내보내기 시, 핀된 썸네일이 쇼츠 영상의 맨 앞에 첫 프레임으로 삽입됩니다.
- 16:9 모드로 전환하면 핀 설정이 자동으로 해제됩니다.

### 생성 중지

생성 진행 중 "⏹ 생성 중지" 버튼을 클릭하면 생성이 즉시 중단됩니다.
- 전체 생성 중에도, 개별 재생성 중에도 중지할 수 있습니다.
- 이미 완성된 썸네일은 유지되며, 아직 생성 중이던 썸네일만 취소됩니다.
- Flow 모드에서는 확장앱에도 취소가 전달되어 완전히 중지됩니다.

### 썸네일 저장 및 내보내기

#### 자동 저장
썸네일이 생성되거나 재생성될 때마다 프로젝트 폴더에 자동으로 저장됩니다.
- 저장 위치: `프로젝트 폴더/썸네일/`
- 파일명: thumbnail_1.png, thumbnail_2.png, thumbnail_3.png, thumbnail_4.png

#### 썸네일 백업
결과 영역 상단의 "💾 썸네일 백업" 버튼을 클릭하면:
1. 저장할 폴더 위치를 선택합니다.
2. 백업 폴더 이름을 입력합니다 (기본값: 프로젝트명_썸네일백업).
3. 모든 썸네일 이미지가 해당 폴더에 복사됩니다.
4. 백업 완료 후 해당 폴더가 자동으로 열립니다.

#### 폴더 열기
"📂 폴더 열기" 버튼을 클릭하면 썸네일이 저장된 폴더가 탐색기에서 바로 열립니다.

### 프로젝트 저장 및 로드

#### 자동 저장 항목
썸네일 관련 데이터는 프로젝트와 함께 자동 저장됩니다:
- 입력한 텍스트 문구 4개
- 참고 이미지 경로 (벤치마크, 추가 이미지)
- 화면 비율 설정
- 핀 상태 (쇼츠 모드)
- 생성된 썸네일 이미지 파일

#### 프로젝트 로드 시 복원
프로젝트를 다시 열면:
- 저장된 텍스트 문구가 입력란에 복원됩니다.
- 참고 이미지가 슬롯에 복원됩니다.
- 생성된 썸네일이 결과 영역에 다시 표시됩니다.
- 핀 상태가 복원됩니다.

### 에러 발생 시 대처법

썸네일 생성 중 발생할 수 있는 오류와 해결 방법:

- "API 키가 설정되지 않았습니다." → 나노2 모드에서 API 키가 없을 때. 설정에서 API 키를 입력하거나, Flow 모드로 전환하세요.
- "대본이 비어있습니다." → 문구 추출 시 대본이 없을 때. 대본을 먼저 작성하세요.
- "생성할 썸네일을 1개 이상 선택해주세요." → 체크된 항목이 없을 때. 최소 1개 이상 체크하세요.
- "모든 썸네일 생성에 실패했습니다." → API 오류 또는 필터링. 잠시 후 다시 시도하세요.
- "API 호출 타임아웃 (120초)" → 네트워크 지연. 네트워크 연결을 확인 후 다시 시도하세요.
- "이미지 응답 없음" → 콘텐츠 필터링. 문구나 참고 이미지를 수정 후 다시 시도하세요.
- "Flow 생성기가 초기화되지 않았습니다." → 확장앱 미연결. 장면/이미지 탭을 먼저 열거나, Chrome 확장앱 연결을 확인하세요.
- "Chrome 확장앱 연결 실패" → 확장앱 미설치 또는 타임아웃. Chrome 확장앱이 설치되어 있는지 확인하고, 다시 시도하세요.
- "해당 썸네일이 아직 생성되지 않았습니다." → 미생성 상태에서 재생성 시도. 먼저 썸네일을 생성하세요.
- "저장된 썸네일이 없습니다." → 백업 시 이미지 없음. 썸네일을 먼저 생성한 후 백업하세요.

참고: 나노2 모드에서는 생성 실패 시 자동으로 최대 3회까지 재시도합니다. Flow 모드에서는 최대 4회까지 재시도합니다.

### 주의사항

1. **문구 길이**: 한 줄에 10자 이내를 권장합니다. 너무 긴 문구는 썸네일에서 가독성이 떨어질 수 있습니다.
2. **참고 이미지 품질**: 벤치마크 썸네일은 실제 유튜브에서 잘 되는 썸네일을 사용하면 더 좋은 결과를 얻을 수 있습니다.
3. **텍스트 없는 썸네일**: 문구를 비우고 생성하면 이미지만으로 구성된 썸네일이 만들어집니다. 이 경우 대본 내용을 참고하여 시각적으로 구성됩니다.
4. **쇼츠 핀과 편집**: 쇼츠 핀을 설정한 경우, [4. 편집] 탭에서 영상 내보내기 시 자동으로 첫 프레임에 삽입됩니다. 핀 설정 여부를 확인하고 내보내기하세요.
5. **대본 수정 후 문구 재추출**: 대본을 수정한 후에는 "대본에서 추출" 버튼을 다시 눌러 문구를 갱신하는 것을 권장합니다.
6. **이미지 형식**: 참고 이미지는 PNG, JPG, JPEG, WebP, BMP 형식을 지원합니다. 다른 형식의 파일은 업로드되지 않습니다.
7. **Flow 모드와 한글 텍스트**: Flow 모드에서도 한글 텍스트 렌더링을 시도하지만, 나노2에 비해 정확도가 낮을 수 있습니다. 한글 텍스트가 중요한 경우 나노2 모드를 추천합니다.

## 구글 Vertex AI JSON 등록 (무료 크레딧 활용)

메이크렌즈는 구글의 AI 기능을 사용하기 위해 API 인증이 필요합니다.
인증 방식은 두 가지가 있으며, 그 중 **Vertex AI JSON 방식**을 사용하면 구글에서 제공하는 **300달러(약 40만원) 무료 크레딧**을 활용하여 무료로 사용할 수 있습니다.

### Vertex AI JSON 방식이 추가된 배경
2026년 3월 10일부터 구글 정책이 변경되어, 신규로 300달러 무료 크레딧을 받은 사용자는 **Gemini 유료 API 키 방식으로는 해당 무료 크레딧을 사용할 수 없게** 되었습니다. 즉, 기존처럼 AI Studio에서 API 키를 발급받아 등록하는 방식으로는 300달러 크레딧이 적용되지 않습니다.
이에 따라 메이크렌즈에서는 **Vertex AI JSON 방식**을 새롭게 추가하여, 변경된 정책 하에서도 300달러 무료 크레딧을 정상적으로 활용할 수 있도록 지원합니다.
하지만 앞으로도 구글정책은 언제든 변경될 수 있으며, 그에 따른 비용은 사용자가 쓴 만큼 구글에 과금될 수 있습니다.

### 영상 가이드
아래 설정 과정을 텍스트로 따라하기 어려우시면, 영상으로 전체 과정을 확인하실 수 있습니다.
영상 링크: https://youtu.be/tbU1uNAQL0Q

### 두 가지 인증 방식 비교

| 구분 | Gemini 유료 API 키 | Vertex AI JSON (권장) |
|------|-------------------|---------------------|
| 비용 | 사용한 만큼 과금 | 300달러 무료 크레딧 사용 가능 |
| 설정 난이도 | 간단 (키 복붙) | 약간의 설정 필요 (이 가이드 참고) |
| 크레딧 기간 | 없음 | 90일간 사용 가능 |
| 크레딧 소진 후 | - | 새 구글 계정으로 재발급 가능 |

*여유가 있고 크레딧을 매번 받기 귀찮으신 분들은, 무료 크레딧 소진 후 유료로 전환하여 계속 사용할 수도 있습니다.*

### 전체 설정 과정 요약

Vertex AI JSON을 등록하려면 아래 5단계를 순서대로 진행합니다:

1단계: 구글 클라우드 300달러 무료 크레딧 받기
2단계: 서비스 계정 만들고 JSON 키 파일 다운로드
3단계: 결제 계정 연결
4단계: Vertex AI API 사용 설정
5단계: 메이크렌즈에서 JSON 파일 등록

*각 단계의 상세 방법은 아래에서 설명합니다.*

### 1단계: 구글 클라우드 300달러 무료 크레딧 받기

구글 클라우드에서는 신규 사용자에게 90일간 사용할 수 있는 300달러 무료 크레딧을 제공합니다.

#### 크레딧 신청 방법

1. 구글에서 "구글 클라우드"를 검색하고, **클라우드 관리 콘솔**에 접속합니다.
2. 화면에 **"무료로 시작하기"** 버튼과 함께 300달러 무료 크레딧 안내가 보이면 클릭합니다.
3. 국가를 **대한민국**으로 확인하고, 하단의 체크박스 두 개를 모두 체크한 뒤 **계속**을 클릭합니다.
4. 계좌유형을 **개인**으로 바꾸고, 주소·이름·우편번호를 입력합니다. (전화번호는 선택사항)
5. **계속**을 누르면 고객정보 입력 화면이 나옵니다. 이름, 주민등록번호, 이동통신사, 전화번호를 입력합니다.
6. 핸드폰으로 받은 인증 코드를 입력하고, 결제수단(카드)을 등록합니다.

#### "무료로 시작하기" 버튼이 보이지 않는 경우
상단의 프로필을 눌러 **다른 구글 계정**으로 전환한 뒤, 다시 구글 클라우드 콘솔에 접속해보세요.

#### 1회성 선불결제에 대해
일부 계정에서는 15,000원 또는 40,000원 등의 1회성 선불결제가 필요할 수 있습니다. 이 금액은 **보증금** 역할이며, 사용하지 않고 해지하면 환불됩니다. 자동 청구는 되지 않으니 걱정하지 않으셔도 됩니다.

#### 크레딧을 다시 받고 싶을 때
크레딧은 구글 계정별로 1회 제공됩니다. 크레딧이 소진되면 새로운 구글 계정을 만들어서 다시 받을 수 있습니다. 이때 되도록 **전화번호와 카드를 기존 계정과 다른 것**으로 사용하는 것을 권장합니다.

### 2단계: 서비스 계정 만들고 JSON 키 파일 다운로드

구글 클라우드에서 메이크렌즈가 사용할 **서비스 계정**을 만들고, 인증용 **JSON 키 파일**을 다운로드합니다.

#### 서비스 계정 생성

1. 구글 클라우드 콘솔 좌측 상단의 **탐색 메뉴(≡)**를 클릭합니다.
2. **IAM 및 관리자** → **서비스 계정**을 클릭합니다.
3. 상단의 **"서비스 계정 만들기"**를 클릭합니다.
4. 계정 이름을 입력합니다 (예: "lens").
5. **만들고 계속하기**를 클릭합니다.
6. **역할 선택**을 클릭하고, 검색창에 **"vertex"**를 입력합니다.
7. 목록에서 **"Vertex AI 사용자"**를 선택합니다.
8. **완료**를 클릭합니다.

#### JSON 키 파일 다운로드

1. 생성된 서비스 계정의 **이메일**을 클릭합니다.
2. 상단의 **키** 탭을 클릭합니다.
3. **키 추가** → **새 키 만들기**를 클릭합니다.
4. 키 유형을 **JSON**으로 선택하고 **만들기**를 클릭합니다.
5. JSON 파일이 컴퓨터에 자동으로 다운로드됩니다.

*이 파일은 한 번만 다운로드 가능하므로 안전한 곳에 보관해주세요.*

#### "보안 사고 방지" 메시지가 뜨는 경우

일부 계정에서 "보안 사고를 방지하기 위해 조직 정책을 적용했습니다"라는 메시지가 뜨면서 키가 생성되지 않을 수 있습니다. 이 경우:

1. 취소 버튼을 눌러 닫습니다.
2. 우측 상단의 **클라우드 셸 활성화(네모 아이콘)**를 클릭합니다.
3. 하단에 뜨는 터미널 창에서 **승인**을 클릭합니다.
4. 안내 영상에서 제공하는 명령어를 복사하여 붙여넣고 실행합니다. 'gcloud projects create 여기만-바꾸세요 --name="MakeLens" --no-enable-cloud-apis' (명령어 중 여기만-바꾸세요 부분만 원하는 이름으로 변경)
5. "Waiting for... Finish done" 메시지가 나오면 성공입니다.
6. 상단의 프로젝트 선택에서 **"조직 없음"**을 클릭하면 새로 만든 프로젝트가 보입니다.
7. 해당 프로젝트를 선택한 뒤, 다시 서비스 계정 → 이메일 → 키 → 키 추가 → 새 키 만들기 → JSON → 만들기 순서로 진행하면 정상적으로 키가 다운로드됩니다.

### 3단계: 결제 계정 연결

서비스 계정이 300달러 무료 크레딧을 사용할 수 있도록 결제 계정을 연결합니다.

1. 좌측 상단 **탐색 메뉴(≡)** → **결제**를 클릭합니다.
2. "결제 계정이 없는 프로젝트"라고 표시되면, 파란색 **"결제 계정 연결"**을 클릭합니다.
3. 결제 계정을 선택한 뒤, **내 결제 계정**을 클릭합니다.
4. **계정 설정**을 클릭합니다.
5. 완료되면 우측 하단에서 남은 크레딧과 사용 기간을 확인할 수 있습니다.
6. 좌측 하단의 **"크레딧"**을 클릭하면 남은 크레딧을 더 자세히 확인할 수 있습니다.

*결제 계정 연결은 반드시 해주셔야 합니다. 연결하지 않으면 API를 사용할 수 없습니다.*

### 4단계: Vertex AI API 사용 설정

Vertex AI 서비스를 활성화해야 메이크렌즈에서 API를 호출할 수 있습니다.

1. 좌측 상단 **탐색 메뉴(≡)** → **API 및 서비스** → **라이브러리**를 클릭합니다.
2. 검색창에 **"Vertex AI"**를 검색합니다.
3. 검색 결과 상단의 **Vertex AI API**를 클릭합니다.
4. **"사용"** 버튼을 클릭하여 API를 활성화합니다.

*이 단계를 빠뜨리면 메이크렌즈에서 API 호출 시 오류가 발생합니다.*

### 5단계: 메이크렌즈에서 JSON 파일 등록

이제 다운로드한 JSON 키 파일을 메이크렌즈에 등록합니다.

#### 등록 방법

1. 메이크렌즈 **설정** 탭에 들어갑니다.
2. 인증 모드 선택에서 **"Vertex AI Json 등록"**을 선택합니다.
3. JSON 파일 등록 영역이 나타납니다.
4. 다음 두 가지 방법 중 하나로 등록합니다:
   - **드래그 앤 드롭**: 다운로드한 JSON 파일을 등록 영역에 끌어다 놓습니다.
   - **파일 찾아보기**: "JSON 파일 찾아보기" 버튼을 클릭하여 파일을 선택합니다.
5. 등록이 완료되면 프로젝트 ID가 표시됩니다.
6. **"API 설정 저장"** 버튼을 클릭하여 저장합니다.

#### 등록 확인 방법

설정 화면에서 **"유료 API 테스트"** 버튼을 클릭합니다. 정상 등록되었다면 강아지 이미지와 함께 "Success" 팝업이 표시됩니다.

### JSON 파일 삭제 및 변경

등록된 JSON 파일을 변경하거나 삭제하려면:
- **파일 변경**: 새로운 JSON 파일을 드래그 앤 드롭하거나 찾아보기로 선택하면 기존 파일이 교체됩니다.
- **파일 삭제**: "초기화" 버튼을 클릭하면 등록된 JSON 정보가 삭제됩니다.

### 인증 모드 전환

메이크렌즈에서는 언제든지 인증 모드를 전환할 수 있습니다:
- 설정 탭의 모드 선택에서 "구글 Gemini 유료 API 키 등록" 또는 "Vertex AI Json 등록"을 선택합니다.
- 모드를 전환해도 기존에 등록한 정보는 유지됩니다.
- 전환 즉시 해당 모드의 인증 방식으로 API가 호출됩니다.
- 선택 된 모드에 따라 작동되니, 유효한 키로 사용자가 잘 체크해야합니다.

### 사용량 확인

Vertex AI 모드에서 사용량과 남은 크레딧을 확인하려면:
- 설정 탭에서 **"사용량 대시보드"** 버튼을 클릭합니다.
- 구글 클라우드 결제 페이지로 이동하여 남은 크레딧과 사용 내역을 확인할 수 있습니다.

### 에러 발생 시 대처법

Vertex AI JSON 사용 중 발생할 수 있는 오류와 해결 방법:

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|-----------|
| 서비스 계정 JSON 파일이 아닙니다. | 잘못된 JSON 파일 등록 | 구글 클라우드에서 "키 추가 → 새 키 만들기 → JSON"으로 받은 파일을 등록하세요. |
| JSON 파일만 등록할 수 있습니다. | .json이 아닌 파일 선택 | .json 확장자의 파일만 등록 가능합니다. |
| JSON 파일을 읽을 수 없습니다. | 파일 손상 또는 접근 불가 | JSON 파일이 정상인지 확인하고, 필요시 다시 다운로드하세요. |
| 먼저 Vertex AI JSON 파일을 등록해주세요. | JSON 미등록 상태에서 테스트 | 설정에서 JSON 파일을 먼저 등록하세요. |
| Vertex AI 모드이지만 서비스 계정 JSON 파일이 설정되지 않았습니다. | JSON 경로 누락 | 설정에서 JSON 파일을 다시 등록하세요. |
| Vertex AI 모드이지만 프로젝트 ID가 설정되지 않았습니다. | 프로젝트 ID 누락 | JSON 파일을 다시 등록하면 프로젝트 ID가 자동으로 설정됩니다. |
| API 테스트 실패 | 결제 계정 미연결 또는 Vertex AI API 미활성화 | 3단계(결제 계정 연결)와 4단계(Vertex AI API 사용 설정)를 확인하세요. |

### 주의사항

1. **JSON 키 파일 보안**: JSON 키 파일에는 비공개 키(private_key)가 포함되어 있으므로, 다른 사람에게 공유하거나 인터넷에 업로드하지 마세요. 서비스 계정의 이메일과 고유 ID는 공유해도 문제없지만, JSON 파일 자체는 절대 공유하면 안 됩니다.
2. **파일 보관**: JSON 키 파일은 한 번만 다운로드됩니다. 분실하면 구글 클라우드에서 새 키를 만들어야 합니다. 안전한 곳에 보관하세요.
3. **크레딧 기간**: 무료 크레딧은 90일간 유효합니다. 기간이 지나면 크레딧이 소멸되므로 기간 내에 사용하세요.
4. **결제 계정 연결 필수**: 3단계의 결제 계정 연결을 반드시 해주셔야 합니다. 연결하지 않으면 크레딧이 있어도 API를 사용할 수 없습니다.
5. **Vertex AI API 활성화 필수**: 4단계의 Vertex AI API 사용 설정을 반드시 해주셔야 합니다. 활성화하지 않으면 API 호출 시 오류가 발생합니다.
6. **유료 전환 시 주의**: 무료 체험 기간이 끝나고 일반 계정을 활성화하면 자동으로 과금됩니다. 유료 전환을 원하지 않으면 일반 계정을 활성화하지 마세요.
7. **설정 영상 참고**: 설정 과정은 구글 정책에 따라 화면이 달라질 수 있습니다. 이 가이드와 실제 화면이 다르면 메이크렌즈에서 제공하는 안내 영상을 함께 참고해주세요. 설정에서 "API 설정 받는 방법" 버튼을 클릭하면 관련 페이지로 이동합니다.

# v1.6.0 업데이트 3가지 기능 추가
## 캡컷 프로젝트 내보내기

메이크렌즈에서 만든 이미지/동영상/TTS를 캡컷(CapCut)으로 바로 보낼 수 있는 기능입니다. 
이미지, 음성, 자막, 타이밍 정보가 그대로 캡컷 프로젝트로 만들어지기 때문에, 캡컷에서 열면 바로 편집을 이어갈 수 있습니다.
특히 저사양 컴퓨터 사용을 하면서, 롱폼 영상에 아주 유용합니다.

내보내기를 하면 캡컷의 프로젝트 목록에 자동으로 등록되며, 캡컷을 열면 바로 해당 프로젝트가 보입니다.
당연히 캡컷이 설치되어있어야합니다.


### 사전 준비

캡컷 내보내기를 하려면 아래 사항들이 모두 완료되어 있어야 합니다:

1. **장면 분석 완료**: [2. 장면/이미지] 탭에서 장면 분석이 실행되어 있어야 합니다.
2. **이미지 생성 완료**: 각 장면의 이미지 또는 비디오가 생성되어 있어야 합니다.
3. **음성 생성 완료**: [3. 음성] 탭에서 음성이 생성되어 있어야 합니다.

하나라도 빠져 있으면 해당 항목을 먼저 완료하라는 안내 메시지가 나타납니다.
또한, 대본/나레이션과 TTS음성이 서로 다르면 자막타이밍이나 장면매칭이 제대로 되지않으니 주의하시기 바랍니다.


### 내보내기 실행 방법

#### 1단계: 내보내기 메뉴 열기

[4. 최종 영상] 탭 상단의 "📤내보내기" 버튼을 클릭하면 메뉴가 열립니다. 
여기서 "🎬 캡컷 프로젝트 내보내기"를 선택합니다.

#### 2단계: 모드 선택 (영상이 있는 경우에만)

만약 비디오 파일이 있으면, 어떤 소스를 사용할지 선택하는 창이 나타납니다:

- **🖼️ 이미지 모드**: 모든 장면을 이미지로 내보냅니다.
- **🎬 자동 모드**: 영상이 있는 장면은 영상으로, 없는 장면은 이미지로 내보냅니다. 
사용 가능한 영상 개수도 함께 표시됩니다.

영상 파일이 없으면 이 단계는 건너뛰고 자동으로 이미지 모드로 진행됩니다.

#### 3단계: 자동 음성 분석

소스 모드를 선택하면 "캡컷 내보내기를 위한 음성 분석 중..."이라는 로딩 메시지와 함께 자동으로 음성 분석이 시작됩니다. 이 과정에서 음성의 타이밍을 분석하고 자막을 생성합니다.

#### 4단계: 캡컷 프로젝트 폴더 확인

음성 분석이 완료되면, 캡컷이 설치된 기본 경로를 자동으로 찾습니다.

- **캡컷이 기본 경로에 설치된 경우**: 자동으로 진행됩니다.
- **기본 경로에 없는 경우**: 캡컷 프로젝트 폴더를 직접 선택하는 창이 나타납니다. 캡컷의 프로젝트 폴더(com.lveditor.draft 폴더)를 찾아서 선택해주세요.

#### 5단계: 프로젝트 파일 생성

진행 상태가 표시되며 3단계로 처리됩니다:
1. "[1/3] 이미지/영상 경로 연결 중..." — 장면별 이미지 또는 영상 파일을 연결합니다.
2. "[2/3] 자막 데이터 구성 중..." — 자막과 타이밍 정보를 구성합니다.
3. "[3/3] 프로젝트 파일 저장 중..." — 캡컷 프로젝트 파일을 저장합니다.

#### 6단계: 완료

내보내기가 완료되면 생성된 프로젝트 폴더 경로가 표시됩니다. 두 가지 버튼이 제공됩니다:

- **"닫기"**: 안내 창을 닫습니다.
- **"캡컷 열기"**: 캡컷을 자동으로 실행합니다. 캡컷이 열리면 프로젝트 목록에서 방금 내보낸 프로젝트를 확인할 수 있습니다.


### 캡컷에서 내보낸 프로젝트 확인하기

내보내기가 완료된 후 캡컷을 열면, 프로젝트 목록에 "{프로젝트이름}_{날짜.시간}" 형태로 새 프로젝트가 나타납니다. 예를 들어 "나의영상_26.03.15.2330" 같은 이름입니다.

프로젝트를 열면 다음 내용이 모두 세팅되어 있습니다:
- 장면별 이미지 또는 영상 클립 (타이밍 자동 맞춤)
- 나레이션 음성
- 자막 (타이밍 자동 동기화)
- 캔버스 크기 (콘텐츠 유형에 맞게 자동 설정: 가로 1920×1080, 세로 1080×1920, 정사각형 1080×1080)


### 자주 묻는 질문 (Q&A)

Q. "프로젝트를 먼저 저장해주세요" 라고 뜨는데요?
A. 프로젝트가 아직 저장되지 않은 상태입니다. 프로젝트를 저장한 후 다시 내보내기를 시도하세요.

Q. "장면이 없습니다" 라고 나와요.
A. 장면 분석이 아직 안 된 상태입니다. [2. 장면/이미지] 탭에서 장면 분석을 먼저 실행하세요.

Q. "이미지가 없습니다" 라고 뜨면요?
A. 이미지가 아직 생성되지 않은 상태입니다. [2. 장면/이미지] 탭에서 이미지를 먼저 생성하세요.

Q. "음성 파일이 없습니다" 라고 나와요.
A. 음성(TTS)이 아직 생성되지 않은 상태입니다. [3. 음성] 탭에서 음성을 먼저 생성하세요.

Q. "캡컷 프로젝트 폴더를 선택해주세요" 라고 뜨는데요?
A. 캡컷이 기본 경로에 설치되지 않아서 자동으로 찾지 못한 것입니다. 캡컷이 설치된 경로에서 프로젝트 폴더(com.lveditor.draft 폴더)를 직접 선택해주세요.

Q. "선택한 폴더에 root_meta_info.json이 없습니다" 라고 뜨면요?
A. 잘못된 폴더를 선택한 것입니다. 캡컷의 프로젝트 폴더인 com.lveditor.draft 폴더를 정확히 선택해주세요.

Q. 내보내기 도중 오류가 발생했어요.
A. 오류 내용을 확인한 후 다시 시도하세요. 문제가 지속되면 지원팀에 문의해주세요.


### 주의사항

1. **원본 파일 이동 금지**: 캡컷 프로젝트는 이미지, 영상, 음성 파일을 복사하지 않고 원래 위치를 참조합니다. 내보내기 후 원본 파일을 이동하거나 삭제하면 캡컷에서 미디어를 찾을 수 없게 됩니다.
2. **캡컷 설치 필요**: PC에 캡컷이 설치되어 있어야 합니다. 캡컷이 없으면 프로젝트 파일은 생성되지만 열 수 없습니다.
3. **효과·전환 없음**: 깨끗한 상태로 내보내기 위해 이미지 효과, 영상 효과, 전환 효과는 모두 제거된 상태로 내보내집니다. 효과가 필요하면 캡컷에서 직접 추가하세요.
4. **번역 ON 상태 확인**: 번역이 ON인 상태에서 내보내기하면 번역된 자막이 적용됩니다. 원본 한국어 자막을 원하면 번역을 OFF로 전환한 후 내보내세요.
5. **동일 프로젝트 반복 내보내기 가능**: 같은 프로젝트를 여러 번 내보내도 날짜와 시간이 포함된 별도의 폴더로 생성되므로, 기존 프로젝트를 덮어쓰지 않습니다.
6. 당연히 캡컷이 설치되어있어야합니다.



## Flow 자동 이미지생성

Google의 Flow(labs.google)를 활용하여 장면별 이미지를 자동으로 생성하는 기능입니다.
기존 Gemini 이미지 생성과 달리 무료이며 나노바나나2의 고품질 이미지를 만들 수 있습니다.
Flow는 Chrome 브라우저의 메이크렌즈 확장앱을 통해 동작합니다.


### 사전 준비 (중요! 반드시 확인하세요)

Flow 이미지 생성을 사용하기 전에 아래 사항을 **반드시** 완료해야 합니다:

1. **메이크렌즈 확장앱 재설치 또는 새로고침 필수**: 이전 버전의 확장앱으로는 Flow 기능이 동작하지 않습니다. 
반드시 확장앱을 재설치하거나 Chrome 확장프로그램 관리(chrome://extensions)에서 새로고침 버튼을 눌러주세요.

2. **Flow 로그인 필수**: Flow를 처음 사용하는 분은 반드시 Google 계정으로 로그인해야 합니다.
   - Flow 페이지: https://labs.google/fx/flow
   - 위 링크에 접속하여 Google 계정으로 로그인하세요.
   - 로그인하지 않으면 이미지 생성 시 "Google 로그인이 필요합니다" 에러가 발생합니다.
   - 봇으로 인한 이미지생성이므로, 만의 하나를 대비해 '서브계정' 계정으로 로그인하시길 권장드립니다. (해당 문제로 계정 차단 시, 메이크렌즈는 책임질 수 없습니다)

3. **브라우저 탭 1개만 유지 (권장)**: 그록 영상생성과 마찬가지로 Flow도 크롬 창을 여러 개 열면 충돌이 발생할 수 있습니다. Flow 이미지 생성시 해당 크롬 창 1개만 열어두세요.


### 그록(Grok) 동영상 생성과의 차이점

Flow와 그록은 동작 방식이 다릅니다. 이 차이를 알아두면 사용할 때 혼동이 없습니다.

**그록 동영상 생성**: 브라우저 화면에서 직접 작업이 진행됩니다. 그래서 그록 탭을 닫거나 다른 페이지로 이동하면 생성이 중단됩니다. 브라우저 화면을 유지해야 합니다.

**Flow 이미지 생성**: 브라우저 **백그라운드(뒤에서)** 작업이 진행됩니다. Flow 탭이 열려 있기만 하면, 그 탭을 보고 있지 않아도 됩니다. 단, Flow 탭 자체를 닫으면 안 됩니다.

정리하면:
- 그록: 브라우저 **화면을 보면서** 대기해야 함
- Flow: 브라우저 **탭만 열어두면** 다른 작업 가능 (탭을 닫지만 않으면 됨)


### 이미지 생성 실행 방법

#### 1단계: 모델 선택

[2. 장면/이미지] 탭에서 이미지 생성 모델을 "Flow"로 선택합니다.

#### 2단계: 이미지 생성 버튼 클릭

이미지 생성 버튼을 클릭하면 자동으로 다음이 진행됩니다:

1. Chrome이 자동으로 실행되고, Flow 페이지(https://labs.google/fx/flow)가 열립니다. (최초에 1번은 반드시 로그인 필요)
2. 확장앱이 자동으로 연결됩니다 (최대 30초 대기).
3. Flow 탭 준비가 완료되면 이미지 생성이 시작됩니다.

#### 3단계: 자동 생성 진행

각 장면의 프롬프트가 자동으로 영어로 변환되고, Flow에 전달되어 이미지가 생성됩니다.
진행 상태가 메이크렌즈 화면에 표시되며, 장면별로 생성이 완료될 때마다 미리보기 이미지가 업데이트됩니다.

- 캐릭터 이미지가 있으면 참조 이미지로 자동 활용됩니다.
- 생성 중 reCAPTCHA 오류 등이 발생하면 자동으로 재시도합니다 (최대 4회).

#### 4단계: 완료

모든 장면의 이미지가 생성되면 완료 메시지가 표시됩니다.


### 자주 묻는 질문 (Q&A)

Q. "Google 로그인이 필요합니다" 라고 뜨는데 어떻게 하나요?
A. Chrome에서 Google 계정에 로그인이 안 된 상태입니다. Flow 페이지(https://labs.google/fx/flow)에 접속해서 Google 계정으로 로그인한 후 다시 시도하세요.

Q. "Google 로그인 인증이 만료되었습니다" 라고 뜨는데요?
A. 로그인 세션이 오래되어 만료된 것입니다. Chrome에서 Google 계정을 다시 로그인해주세요.

Q. "Flow 보안 인증에 실패했습니다" (reCAPTCHA) 에러가 나요.
A. 짧은 시간에 이미지를 너무 많이 생성하면 Google이 자동화로 의심해서 발생합니다. Chrome에서 Flow 탭을 닫고 다시 열거나, Chrome을 재시작한 후 1~2분 기다렸다가 다시 시도하세요.

Q. "Flow 요청 한도를 초과했습니다" 라고 뜨면요?
A. Flow는 무료 서비스라 시간당 생성 횟수에 제한이 있습니다. 최소 1~2분 기다려주세요. 그래도 안 되면 10~15분 더 기다려주세요. 급하면 다른 모델(Gemini 등)로 변경해서 생성하세요.

Q. "확장앱이 연결되지 않았습니다" 라고 나와요.
A. 메이크렌즈 확장앱이 Chrome에서 제대로 동작하지 않는 상태입니다. Chrome이 실행 중인지 확인하고, 확장앱을 새로고침하거나 Chrome을 재시작하세요.

Q. "이미지 생성이 거부되었습니다" 라고 뜨는데 왜 그런가요?
A. Google AI 안전 정책에 의해 차단된 것입니다. 프롬프트에 유명인 이름, 폭력적 표현, 선정적 키워드, 특정 브랜드명 등이 포함되어 있으면 생성이 거부됩니다. 프롬프트를 수정해주세요. 프로그램 문제가 아니라 Google 정책입니다.

Q. "Flow 브라우저 준비에 실패했습니다" 라고 뜨면요?
A. Chrome에서 Flow 탭(labs.google)이 열려있는지, Google 계정에 로그인되어 있는지 확인하세요. 확인 후에도 안 되면 Chrome을 재시작하세요.


### 주의사항

1. **확장앱 업데이트 필수**: 이전 버전 확장앱에는 Flow 기능이 없습니다. 반드시 새로고침 또는 재설치하세요.
2. **처음 사용 시 Flow 로그인 필수**: https://labs.google/fx/flow 에 접속하여 Google 계정으로 로그인하세요.(세컨계정 권장)
3. **브라우저 탭 1개 권장**: 크롬 창을 1개만 유지하세요. 여러 창을 열면 충돌이 발생할 수 있습니다.(그록도 마찬가지)
4. **Flow 탭 닫지 마세요**: 이미지 생성 중 Flow 탭을 닫으면 생성이 중단됩니다.
5. **순차 처리**: Flow는 한 번에 하나씩 장면을 순서대로 생성합니다. 병렬(동시) 생성은 지원하지 않습니다.
6. **무료 서비스 제한**: Flow는 Google의 무료 서비스이므로, 시간당 생성 가능한 이미지 수에 제한이 있을 수 있습니다. 한도 초과 시 잠시 기다렸다가 다시 시도하세요. 또한 구글 정책변경으로 언제든 이 서비스는 종료될 수 있습니다.



## 장면/이미지 프롬프트 불러오기

장면 백업 파일(장면.txt)에서 이미지 프롬프트와 동영상 프롬프트를 불러와서, 현재 장면에 한번에 적용하는 기능입니다.

예를 들어, 장면 분석 후 생성된 프롬프트가 마음에 안 들 때, 장면.txt 파일을 메모장 등으로 열어서 프롬프트를 원하는 대로 수정한 뒤 다시 불러오면 됩니다. 장면이 많을수록 하나씩 수정하는 것보다 훨씬 편리합니다.
따로 사용중인 AI가 있으시면, AI에게 더 나은 프롬프트를 요청해보세요.


### 사전 준비

1. **장면 분석이 완료되어 있어야 합니다**: 장면이 없으면 프롬프트를 불러올 곳이 없으므로, 먼저 장면 분석을 실행하세요.

2. **장면 백업 파일(장면.txt)이 필요합니다**: 프롬프트를 불러올 .txt 파일이 있어야 합니다. 장면 분석을 하면 프로젝트 폴더 안에 `장면/장면.txt` 파일이 자동으로 생성됩니다. 또는 `💾 저장/열기` → `🧾 장면 백업`으로 수동 백업할 수도 있습니다.


### 사용 방법

#### 1단계: 장면 백업 파일 준비

먼저 프로젝트 폴더 안의 `장면/장면.txt` 파일을 메모장이나 텍스트 편집기로 엽니다.
파일을 열면 아래와 같은 형식으로 되어 있습니다:

```
==========
장면 1

[나레이션]
여기에 나레이션 텍스트...

[이미지 프롬프트]
여기에 이미지 프롬프트...

[동영상 프롬프트]
여기에 동영상 프롬프트...

==========
장면 2

[나레이션]
...
```

이 파일에서 `[이미지 프롬프트]` 아래의 내용이나 `[동영상 프롬프트]` 아래의 내용을 원하는 대로 수정한 뒤 저장합니다.

#### 2단계: 프롬프트 불러오기 실행

[2. 장면/이미지] 탭에서 `📊 장면 분석 실행` 버튼을 클릭하면 선택 팝업이 나타납니다.
여기서 `📂 프롬프트 불러오기 (txt)` 버튼을 클릭합니다.

#### 3단계: 파일 선택

파일 선택 창이 열리면 수정한 장면.txt 파일을 선택합니다.

#### 4단계: 적용 및 완료

파일을 선택하면 자동으로 프롬프트가 적용됩니다.
완료되면 "이미지 프롬프트: N개 적용, 동영상 프롬프트: M개 적용" 메시지가 표시됩니다.
적용된 프롬프트는 즉시 장면 카드에 반영되고, 프로젝트에 자동 저장됩니다.


### 이미지 프롬프트와 동영상 프롬프트

하나의 장면.txt 파일 안에 이미지 프롬프트와 동영상 프롬프트가 함께 들어있습니다.

- **이미지 프롬프트만 수정**한 경우 → 이미지 프롬프트만 적용됩니다.
- **동영상 프롬프트만 수정**한 경우 → 동영상 프롬프트만 적용됩니다.
- **둘 다 수정**한 경우 → 둘 다 한번에 적용됩니다.

비어있는 프롬프트는 건너뛰므로, 수정하지 않은 부분은 기존 내용이 유지됩니다.


### 장면 수가 다를 때

대본을 수정해서 장면 분석을 다시 했더니 장면 수가 바뀌었는데, 이전에 백업한 장면.txt를 불러오고 싶은 경우가 있을 수 있습니다.

이 경우 "현재 장면: N개, 파일 장면: M개" 라는 경고 메시지가 나타납니다.
"확인"을 누르면 **장면 번호가 일치하는 것만** 적용됩니다.

예시:
- 현재 장면: 1~8번 (8개)
- 파일 장면: 1~10번 (10개)
- → 1~8번 장면의 프롬프트만 적용되고, 9~10번은 무시됩니다.


### 이런 경우에는 불러오기가 안 됩니다

- **장면 분석을 아직 안 한 경우**: "장면이 없습니다. 먼저 장면 분석을 실행하세요." 메시지가 나타납니다.
- **파일 형식이 맞지 않는 경우**: 장면.txt 형식이 아닌 일반 텍스트 파일을 선택하면 "파일에서 장면 프롬프트를 찾을 수 없습니다." 메시지가 나타납니다. 반드시 `==========`, `장면 N`, `[이미지 프롬프트]` 형식이 포함된 파일이어야 합니다.
- **파일 선택을 취소한 경우**: 아무 일도 일어나지 않습니다.


### 주의사항

1. **불러오기 전에 장면 백업을 먼저 하세요**: 프롬프트 불러오기는 기존 프롬프트를 덮어씁니다. 혹시 되돌리고 싶을 수 있으니, 불러오기 전에 `💾 저장/열기` → `🧾 장면 백업`으로 현재 상태를 백업해두는 것을 권장합니다.
2. **파일은 반드시 장면.txt 형식이어야 합니다**: 메이크렌즈가 생성한 장면.txt 형식을 그대로 유지하면서 프롬프트 부분만 수정하세요. 구분자(`==========`)나 섹션 헤더(`[이미지 프롬프트]`)를 지우면 인식되지 않습니다.
3. **적용 후 자동 저장됩니다**: 불러오기가 완료되면 프로젝트에 자동 저장되므로, 별도로 저장 버튼을 누를 필요가 없습니다.
4. 스타일 프롬프트를 A로 설정하고 장면분석실행 후, B 스타일 프롬프트로 변경하거나, 다른 스타일의 이미지프롬프트로 입력한 경우 최초의 A스타일과 충돌이 일어나거나, 중복 또는 A스타일로 적용될 수있습니다. 되도록 스타일은 유지하시고, 스타일도 변경하고 싶다면, 장면분석실행을 다시 하고나서 이미지 프롬프트를 교체해주세요.
